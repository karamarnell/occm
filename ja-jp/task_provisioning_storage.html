    プロビジョニングストレージ GitHubで編集 ドキュメントの変更をリクエストする

    06/21/2019 貢献者

    ボリュームとアグリゲートを管理することで、Cloud ManagerからCloud Volumes ONTAPシステム用の追加のNFSおよびCIFSストレージをプロビジョニングできます。 iSCSIストレージを作成する必要がある場合は、System Managerから作成する必要があります。

    すべてのディスクとアグリゲートを作成してCloud Managerから直接削除する必要があります。他の管理ツールからこれらの操作を実行しないでください。これを行うと、システムの安定性に影響を与え、将来的にディスクを追加する能力を妨げ、冗長なクラウドプロバイダー料金を発生させる可能性があります。

    プロビジョニングボリューム

    Cloud Volumes ONTAPシステムを起動した後にさらにストレージが必要な場合は、Cloud Managerから新しいNFSおよびCIFSボリュームをプロビジョニングできます。

    あなたが始める前に

    AWSでCIFSを使用したい場合は、DNSとActive Directoryを設定しておく必要があります。詳細については、「 Cloud Volumes ONTAP for AWSのネットワーク要件」を参照してください。

    ステップ

    1. [作業環境]ページで、ボリュームをプロビジョニングするCloud Volumes ONTAPシステムの名前をダブルクリックします。

    2. 任意のアグリゲートまたは特定のアグリゲートに新しいボリュームを作成します。

      アクションステップ

      新しいボリュームを作成し、Cloud Managerに包含アグリゲートを選択させる

      [ 新しいボリュームを追加]をクリックします。

      特定のアグリゲートに新しいボリュームを作成する

      1. メニューアイコンをクリックしてから、[ 詳細設定 ] > [詳細割り当て ]をクリックします。

      2. 集計のメニューをクリックします。

      3. ボリュームの作成をクリックします 。

    3. 新しいボリュームの詳細を入力して、[ 続行 ]をクリックします。

      このページのいくつかのフィールドは一目瞭然です。次の表は、ガイダンスが必要となる可能性があるフィールドについて説明しています。

      フィールド説明

      サイズ

      入力できる最大サイズは、シンプロビジョニングを有効にしているかどうかによって大きく異なります。これにより、現在利用可能な物理ストレージよりも大きいボリュームを作成できます。

      アクセス制御（NFSのみ）

      エクスポートポリシーは、ボリュームにアクセスできるサブネット内のクライアントを定義します。デフォルトでは、Cloud Managerはサブネット内のすべてのインスタンスへのアクセスを提供する値を入力します。

      権限とユーザー/グループ（CIFSのみ）

      これらのフィールドを使用すると、ユーザーおよびグループの共有へのアクセスレベル（アクセス制御リストまたはACLとも呼ばれます）を制御できます。ローカルまたはドメインのWindowsユーザーまたはグループ、あるいはUNIXユーザーまたはグループを指定できます。ドメインのWindowsユーザー名を指定する場合は、domain \ usernameの形式でユーザーのドメインを含める必要があります。

      スナップショットポリシー

      Snapshotコピーポリシーは、自動的に作成されたNetApp Snapshotコピーの頻度と数を指定します。 NetApp Snapshotコピーは、パフォーマンスへの影響がなく、最小限の記憶域しか必要としないポイントインタイムファイルシステムイメージです。デフォルトのポリシーを選択することも、noneを選択することもできます。一時データにはnoneを選択することができます。たとえば、Microsoft SQL Serverの場合はtempdbです。

    4. CIFSプロトコルを選択し、CIFSサーバが設定されていない場合は、[Create a CIFS Server]ダイアログボックスでサーバの詳細を指定し、[ Save ]をクリックして続行します 。

      フィールド説明

      DNSのプライマリおよびセカンダリIPアドレス

      CIFSサーバの名前解決を提供するDNSサーバのIPアドレス。

      表示されたDNSサーバーには、CIFSサーバーが参加するドメインのActive Directory LDAPサーバーとドメインコントローラーを見つけるために必要なサービスロケーションレコード（SRV）が含まれている必要があります。

      参加するActive Directoryドメイン

      CIFSサーバを参加させたいActive Directory（AD）ドメインのFQDN。

      ドメインに参加することを承認された資格情報

      ADドメイン内の指定された組織単位（OU）にコンピューターを追加するのに十分な特権を持つWindowsアカウントの名前とパスワード。

      CIFSサーバのNetBIOS名

      ADドメイン内で一意のCIFSサーバ名。

      組織ユニット

      ADドメイン内のCIFSサーバーに関連付ける組織単位。デフォルトはCN = Computersです。

      DNSドメイン

      Cloud Volumes ONTAPストレージ仮想マシン（SVM）のDNSドメイン。ほとんどの場合、ドメインはADドメインと同じです。

      NTPサーバ

      Active Directory DNSを使用してNTPサーバーを構成するには、 Active Directoryドメインを使用を選択します。別のアドレスを使用してNTPサーバーを構成する必要がある場合は、APIを使用してください。詳細については、 Cloud Manager API開発者ガイドを参照してください。

    5. [Usage Profile]、[Disk Type]、および[Tiering Policy]ページで、ストレージ効率化機能を有効にするかどうかを選択し、必要に応じてディスクタイプを選択し、階層化ポリシーを編集します。

      ヘルプについては、以下を参照してください。

      • ボリューム使用量プロファイルについて

      • AWSでシステムのサイズを設定する

      • Azureでシステムのサイズを変更する

      • データ階層化の概要

    6. Goをクリックしてください。

    結果

    クラウドボリュームONTAPはボリュームをプロビジョニングします。

    終了したら

    CIFS共有をプロビジョニングした場合は、ユーザーまたはグループにファイルとフォルダへのアクセス許可を付与し、それらのユーザーが共有にアクセスしてファイルを作成できることを確認します。

    クォータをボリュームに適用する場合は、システムマネージャまたはCLIを使用する必要があります。割り当て制限を使用すると、ユーザー、グループ、またはqtreeが使用するディスク容量とファイル数を制限または追跡できます。

    HA構成の2番目のノードにボリュームをプロビジョニングする

    デフォルトでは、Cloud ManagerはHA構成の最初のノードにボリュームを作成します。両方のノードがクライアントにデータを提供するアクティブ - アクティブ構成が必要な場合は、2番目のノードにアグリゲートとボリュームを作成する必要があります。

    ステップ

    1. [Working Environments]ページで、アグリゲートを管理するCloud Volumes ONTAPの作業環境の名前をダブルクリックします。

    2. メニューアイコンをクリックしてから、[ 詳細設定 ] > [詳細割り当て ]をクリックします。

    3. [ Add Aggregate]をクリックしてからアグリゲートを作成します。

    4. Home Nodeには、HAペアの2番目のノードを選択します。

    5. Cloud Managerがアグリゲートを作成したら、それを選択して[ Create volume ]をクリックします 。

    6. 新しいボリュームの詳細を入力して、[ 作成 ]をクリックします 。

    終了したら

    必要に応じて、このアグリゲートに追加のボリュームを作成できます。

    複数のAWSアベイラビリティゾーンにデプロイされたHAペアの場合、ボリュームが存在するノードの浮動IPアドレスを使用して、ボリュームをクライアントにマウントする必要があります。

    集計を作成する

    自分でアグリゲートを作成することも、ボリュームを作成するときにCloud Managerに自動的に作成させることもできます。自分でアグリゲートを作成する利点は、基礎となるディスクサイズを選択できることです。これにより、必要な容量またはパフォーマンスに合わせてアグリゲートのサイズを設定できます。

    ステップ

    1. [作業環境]ページで、アグリゲートを管理したいCloud Volumes ONTAPインスタンスの名前をダブルクリックします。

    2. メニューアイコンをクリックしてから、[ 詳細設定 ] > [詳細割り当て ]をクリックします。

    3. [ Add Aggregate]をクリックしてから、アグリゲートの詳細を指定します。

      ディスクタイプとディスクサイズについては、「 設定の計画 」を参照してください 。

    4. Go]をクリックし、 承認し、購入をクリックします。

    iSCSI LUNのプロビジョニング

    iSCSI LUNを作成したい場合は、System Managerから作成する必要があります。

    あなたが始める前に

    • Host Utilitiesは、LUNに接続するホストにインストールして設定する必要があります。

    • ホストからiSCSIイニシエータ名を記録しておく必要があります。 LUNのigroupを作成するときにこの名前を指定する必要があります。

    • System Managerでボリュームを作成する前に、十分なスペースを持つアグリゲートがあることを確認する必要があります。 Cloud Managerでアグリゲートを作成する必要があります。詳細については、 集計の作成を参照してください。

    このタスクについて

    これらの手順では、バージョン9.3以降でSystem Managerを使用する方法について説明します。

    ステップ

    1. System Managerにログインします。

    2. [ ストレージ]> [LUN]をクリックします 。

    3. [ 作成]をクリックし、プロンプトに従ってLUNを作成します。

    4. ホストからLUNに接続します。

      手順については、オペレーティングシステムのHost Utilitiesのマニュアルを参照してください。
