    既存のストレージを管理する GitHubで編集 ドキュメントの変更をリクエストする

    05/09/2019 貢献者

    Cloud Managerを使用すると、ボリューム、アグリゲート、およびCIFSサーバーを管理できます。また、容量の問題を回避するためにボリュームを移動するように求められます。

    既存のボリュームを管理する

    ストレージのニーズが変化したときに既存のボリュームを管理できます。ボリュームを表示、編集、複製、復元、および削除できます。

    ステップ

    1. [作業環境]ページで、ボリュームを管理するCloud Volumes ONTAPの作業環境をダブルクリックします。

    2. ボリュームを管理します。

      仕事アクション

      ボリュームに関する情報を表示する

      ボリュームを選択し、[ 情報 ]をクリックします。

      ボリュームを編集する（読み書き可能なボリュームのみ）

      1. ボリュームを選択して、[ 編集 ]をクリックします。

      2. ボリュームのスナップショットポリシー、NFSアクセス制御リストを変更するか、アクセス許可を共有し、[ 更新 ]をクリックします 。

      ボリュームを複製する

      1. ボリュームを選択して、[ クローン作成 ]をクリックします。

      2. 必要に応じてクローン名を変更し、[ 複製 ] をクリックします。

      このプロセスによってFlexCloneボリュームが作成されます。 FlexCloneボリュームは書き込み可能なポイントインタイムコピーで、メタデータ用に少量のスペースを使用し、データが変更または追加されたときにのみ追加スペースを消費するためスペース効率がよくなります。

      FlexCloneボリュームの詳細については、 『 ONTAP 9論理ストレージ管理ガイド』を参照してください。

      Snapshotコピーから新しいボリュームにデータを復元する

      1. ボリュームを選択し、[ スナップショットコピーから復元]をクリックします 。

      2. Snapshotコピーを選択し、新しいボリュームの名前を入力してから、[ 復元 ]をクリックします。

      Snapshotコピーをオンデマンドで作成する

      1. ボリュームを選択し、[ スナップショットコピーの作成 ]をクリックします 。

      2. 必要に応じて名前を変更し、[ 作成 ]をクリックします 。

      NFSマウントコマンドを取得する

      1. ボリュームを選択して、[ コマンドのマウント ]をクリックします。

      2. コピーをクリックします。

      基礎となるディスクの種類を変更する

      1. ボリュームを選択し、[ ディスクタイプと階層化ポリシーの変更 ]をクリックします。

      2. ディスクの種類を選択し、[ 変更 ]をクリックします。

      Cloud Managerは、選択したディスクタイプを使用する既存のアグリゲートにボリュームを移動するか、またはボリュームの新しいアグリゲートを作成します。

      階層化ポリシーを変更する

      1. ボリュームを選択し、[ ディスクタイプと階層化ポリシーの変更 ]をクリックします。

      2. ポリシーの編集をクリックします。

      3. 別のポリシーを選択して[ 変更 ]をクリックします。

      Cloud Managerは、選択したディスクタイプを階層化して使用する既存のアグリゲートにボリュームを移動するか、またはボリュームの新しいアグリゲートを作成します。

      ボリュームのS3への同期を有効または無効にする

      ボリュームを選択して、[ S3に同期 ]または[同期関係の削除 ]をクリックします。

      これらのオプションを使用する前に、S3への同期機能を有効にする必要があります。手順については、 AWS S3へのデータの同期を参照してください。

      ボリュームを削除する

      1. ボリュームを選択し、[ 削除 ]をクリックします。

      2. 確認のためにもう一度[ 削除]をクリックします。

    既存の集計を管理する

    ディスクを追加し、アグリゲートに関する情報を表示し、それらを削除することによって、アグリゲートを自分で管理します。

    あなたが始める前に

    アグリゲートを削除したい場合は、最初にアグリゲート内のボリュームを削除しておく必要があります。

    このタスクについて

    アグリゲートのスペースが不足している場合は、OnCommand System Managerを使用してボリュームを別のアグリゲートに移動できます。

    ステップ

    1. [作業環境]ページで、アグリゲートを管理したいCloud Volumes ONTAP作業環境をダブルクリックします。

    2. メニューアイコンをクリックしてから、[ 詳細設定 ] > [詳細割り当て ]をクリックします。

    3. 集計を管理します。

      仕事アクション

      集合体に関する情報を表示する

      集計を選択して[ 情報 ]をクリックします。

      特定のアグリゲートにボリュームを作成する

      アグリゲートを選択して[ ボリュームの作成 ]をクリックします 。

      アグリゲートにディスクを追加する

      1. アグリゲートを選択し、[ AWSディスクの追加 ]または[ Azureディスクの追加 ] をクリックします 。

      2. 追加するディスクの数を選択して[ 追加 ] をクリックします。

        アグリゲート内のすべてのディスクは同じサイズでなければなりません。

      集計を削除する

      1. ボリュームを含まないアグリゲートを選択して、[ 削除]をクリックします。

      2. 確認のためにもう一度[ 削除]をクリックします。

    CIFSサーバの修正

    DNSサーバーまたはActive Directoryドメインを変更した場合は、Cloud Volumes ONTAPでCIFSサーバーを変更して、引き続きクライアントにストレージを提供できるようにする必要があります。

    ステップ

    1. 作業環境から、メニューアイコンをクリックし、次にAdvanced> CIFS setupの順にクリックします 。

    2. CIFSサーバーの設定を指定します。

      仕事アクション

      DNSのプライマリおよびセカンダリIPアドレス

      CIFSサーバの名前解決を提供するDNSサーバのIPアドレス。

      表示されたDNSサーバーには、CIFSサーバーが参加するドメインのActive Directory LDAPサーバーとドメインコントローラーを見つけるために必要なサービスロケーションレコード（SRV）が含まれている必要があります。

      参加するActive Directoryドメイン

      CIFSサーバを参加させたいActive Directory（AD）ドメインのFQDN。

      ドメインに参加することを承認された資格情報

      ADドメイン内の指定された組織単位（OU）にコンピューターを追加するのに十分な特権を持つWindowsアカウントの名前とパスワード。

      CIFSサーバのNetBIOS名

      ADドメイン内で一意のCIFSサーバ名。

      組織ユニット

      ADドメイン内のCIFSサーバーに関連付ける組織単位。デフォルトはCN = Computersです。

      DNSドメイン

      Cloud Volumes ONTAPストレージ仮想マシン（SVM）のDNSドメイン。ほとんどの場合、ドメインはADドメインと同じです。

      NTPサーバ

      Active Directory DNSを使用してNTPサーバーを構成するには、 Active Directoryドメインを使用を選択します。別のアドレスを使用してNTPサーバーを構成する必要がある場合は、APIを使用してください。詳細については、 Cloud Manager API開発者ガイドを参照してください。

    3. 保存をクリックします 。

    結果

    Cloud Volumes ONTAPはCIFSサーバを変更内容で更新します。

    容量の問題を回避するためにボリュームを移動する

    Cloud Managerは、容量の問題を回避するためにボリュームの移動が必要であることを示すAction Requiredメッセージを表示することがありますが、問題を修正するための推奨を提供することはできません。このような場合は、問題を解決する方法を特定してから、1つ以上のボリュームを移動する必要があります。

    ステップ

    1. 問題を修正する方法を特定します 。

    2. 分析に基づいて、容量の問題を回避するためにボリュームを移動します。

      • ボリュームを別のシステムに移動します 。

      • 同じシステム上の別のアグリゲートにボリュームを移動します 。

    容量の問題を修正する方法の特定

    容量の問題を回避するためにCloud Managerがボリュームの移動を推奨できない場合は、移動する必要があるボリュームと、それらを同じシステム上の別のアグリゲートに移動するか、別のシステムに移動するかを指定する必要があります。

    ステップ

    1. 対処が必要なメッセージで詳細情報を表示して、容量制限に達したアグリゲートを特定してください。

      たとえば、詳細情報では、次のようになります。集計aggr1が容量制限に達しました。

    2. アグリゲートから移動する1つ以上のボリュームを識別します。

      1. 作業環境で、メニューアイコンをクリックしてから、[ 詳細設定 ] > [詳細割り当て ]をクリックします。

      2. 集計を選択して、[ 情報 ]をクリックします。

      3. ボリュームの一覧を展開します。

      4. 各ボリュームのサイズを確認し、1つ以上のボリュームを選択してアグリゲートから移動します。

        将来追加の容量の問題を回避するために、アグリゲート内のスペースを解放するのに十分な大きさのボリュームを選択する必要があります。

    3. システムがディスク制限に達していない場合は、ボリュームを同じシステム上の既存のアグリゲートまたは新しいアグリゲートに移動する必要があります。

      詳しくは、 容量の問題を回避するためのボリュームの別のアグリゲートへの移動を参照してください。

    4. システムがディスク制限に達した場合は、次のいずれかを実行します。

      1. 未使用のボリュームを削除します。

      2. アグリゲート上の空きスペースにボリュームを再配置します。

        詳しくは、 容量の問題を回避するためのボリュームの別のアグリゲートへの移動を参照してください。

      3. 2つ以上のボリュームをスペースのある別のシステムに移動します。

        詳しくは、 容量の問題を回避するためのボリュームの別のシステムへの移動を参照してください。

    容量の問題を回避するためにボリュームを別のシステムに移動する

    容量の問題を回避するために、1つ以上のボリュームを別のCloud Volumes ONTAPシステムに移動できます。システムがそのディスク制限に達した場合、これを行う必要があるかもしれません。

    このタスクについて

    このタスクの手順に従って、次の対処が必要なメッセージを修正できます。 

    容量の問題を回避するには、ボリュームを移動する必要があります。ただし、システムがディスクの制限に達したため、Cloud Managerはこのアクションを実行できません。
    ステップ

    1. 利用可能な容量を持つCloud Volumes ONTAPシステムを特定するか、または新しいシステムを展開します。

    2. ソース作業環境をターゲット作業環境にドラッグアンドドロップして、ボリュームのワンタイムデータ複製を実行します。

      詳しくは、 システム間でのデータの複製を参照してください。

    3. [Replication Status]ページに移動し、SnapMirror関係を解除して、複製ボリュームをデータ保護ボリュームから読み取り/書き込みボリュームに変換します。

      詳しくは、 データ複製スケジュールおよび関係の管理を参照してください。

    4. データアクセス用にボリュームを設定します。

      データアクセス用のデスティネーションボリュームの設定については、 『 ONTAP 9 Volume Disaster Recovery Expressガイド』を参照してください 。

    5. 元のボリュームを削除します。

      詳しくは、 既存ボリュームの管理を参照してください。

    容量の問題を回避するためにボリュームを別の集合に移動する

    容量の問題を回避するために、1つ以上のボリュームを別のアグリゲートに移動できます。

    このタスクについて

    このタスクの手順に従って、次の対処が必要なメッセージを修正できます。 

    容量の問題を回避するには、2つ以上のボリュームを移動する必要があります。ただし、Cloud Managerはこのアクションを実行できません。
    ステップ

    1. 既存のアグリゲートに、移動する必要があるボリュームに使用可能な容量があるかどうかを確認します。

      1. 作業環境で、メニューアイコンをクリックしてから、[ 詳細設定 ] > [詳細割り当て ]をクリックします。

      2. 各アグリゲートを選択し、[ 情報 ]をクリックして、使用可能容量（アグリゲート容量 - 使用済みアグリゲート容量）を表示します。

    2. 必要に応じて、既存のアグリゲートにディスクを追加します。

      1. アグリゲートを選択し、[ ディスクの追加 ] をクリックします 。

      2. 追加するディスクの数を選択して、[ 追加 ] をクリックします。

    3. 利用可能なキャパシティがアグリゲートにない場合は、新しいアグリゲートを作成してください。

      詳細については、 集計の作成を参照してください。

    4. System ManagerまたはCLIを使用してボリュームをアグリゲートに移動します。

    5. ほとんどの場合、System Managerを使ってボリュームを移動できます。

      手順については、 『 ONTAP 9 Volume Move Expressガイド』を参照してください。
