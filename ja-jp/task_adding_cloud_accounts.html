    Cloud Managerへのクラウドプロバイダーアカウントの追加 GitHubで編集 ドキュメントの変更をリクエストする

    07/10/2019 貢献者

    Cloud Volumes ONTAPを別のクラウドアカウントにデプロイする場合は、それらのアカウントに必要な権限を付与してから、その詳細をCloud Managerに追加する必要があります。

    Cloud CentralからCloud Managerをデプロイすると、Cloud Managerは、Cloud Managerをデプロイしたアカウントにクラウドプロバイダーアカウントを自動的に追加します。既存のシステムにCloud Managerソフトウェアを手動でインストールした場合、初期クラウドプロバイダーアカウントは追加されません。

    AWSアカウントをセットアップしてCloud Managerに追加する

    Cloud Volumes ONTAPを異なるAWSアカウントにデプロイする場合は、それらのアカウントに必要な権限を付与してから、その詳細をCloud Managerに追加する必要があります。アクセス許可をどのように提供するかは、Cloud ManagerにAWSキーを提供するのか、信頼できるアカウントのロールのARNを提供するのかによって異なります。

    • AWSキーを提供するときに権限を付与する

    • 他のアカウントでIAMロールを引き受けることによる権限の付与

    AWSキーを提供するときに権限を付与する

    IAMユーザーのAWSキーをCloud Managerに提供する場合は、そのユーザーに必要な権限を付与する必要があります。 Cloud ManagerのIAMポリシーは、Cloud Managerが使用を許可されているAWSのアクションとリソースを定義します。

    ステップ

    1. Cloud Managerの[Policies]ページからCloud ManagerのIAMポリシーをダウンロードします 。

    2. IAMコンソールから、Cloud ManagerのIAMポリシーのテキストをコピーして貼り付けることによって、独自のポリシーを作成します。

      AWSドキュメント：IAMポリシーの作成

    3. ポリシーをIAMロールまたはIAMユーザーに添付します。

      • AWSドキュメント：IAMロールの作成

      • AWSドキュメント：IAMポリシーの追加と削除

    結果

    アカウントに必要な権限が付与されました。 これでCloud Managerに追加できます 。

    他のアカウントでIAMロールを引き受けることによる権限の付与

    IAMロールを使用して、Cloud ManagerインスタンスをデプロイしたソースAWSアカウントと他のAWSアカウントの間に信頼関係を設定できます。その後、信頼済みアカウントからIAMロールのARNをCloud Managerに提供します。

    ステップ

    1. Cloud Volumes ONTAPをデプロイするターゲットアカウントに移動し、[ 別のAWSアカウント]を選択してIAMロールを作成します 。

      次のことを必ず行ってください。

      • Cloud Managerインスタンスが存在するアカウントのIDを入力してください。

      • Cloud Managerの[Policies]ページから利用できるCloud ManagerのIAMポリシーを添付します 。

    2. Cloud Managerインスタンスが存在するソースアカウントに移動し、インスタンスに関連付けられているIAMロールを選択します。

      1. 信頼関係>信頼関係の編集をクリックします 。

      2. 「sts：AssumeRole」アクションと、作成したロールのARNをターゲットアカウントに追加します。

        例 

         { "Version": "2012-10-17", "Statement": { "Effect": "Allow", "Action": "sts:AssumeRole", "Resource": "arn:aws:iam::ACCOUNT-B-ID:role/ACCOUNT-B-ROLENAME" } }
    結果

    アカウントに必要な権限が付与されました。 これでCloud Managerに追加できます 。

    Cloud ManagerへのAWSアカウントの追加

    必要な権限を持つAWSアカウントを指定したら、そのアカウントをCloud Managerに追加できます。これにより、そのアカウントでCloud Volumes ONTAPシステムを起動できます。

    ステップ

    1. Cloud Managerコンソールの右上にある[設定]アイコンをクリックし、[ クラウドプロバイダとサポートアカウント]を選択します 。

    2. [ 新しいアカウントを追加]をクリックして[ AWS ]を選択します。

    3. AWSキーまたは信頼されたIAMロールのARNを提供するかどうかを選択してください。

    4. ポリシーの要件が満たされていることを確認してから、[ アカウントの作成]をクリックします 。

    結果

    新しい作業環境を作成するときに、Details and Credentialsページから別のアカウントに切り替えることができます。

    AzureアカウントをセットアップしてCloud Managerに追加する

    Cloud Volumes ONTAPを異なるAzureアカウントにデプロイする場合は、それらのアカウントに必要な権限を付与してから、そのアカウントに関する詳細をCloud Managerに追加する必要があります。

    • サービスプリンシパルを使用してAzureアクセス許可を付与する

    • Cloud ManagerにAzureアカウントを追加する

    サービスプリンシパルを使用してAzureアクセス許可を付与する

    Cloud Managerには、Azureでアクションを実行するための権限が必要です。 Azure Active Directoryでサービスプリンシパルを作成および設定し、Cloud Managerに必要なAzure認証情報を取得することで、Azureアカウントに必要なアクセス許可を付与できます。

    このタスクについて

    次の図は、Cloud ManagerがAzureで操作を実行するためのアクセス許可を取得する方法を示しています。 1つ以上のAzureサブスクリプションに関連付けられているサービスプリンシパルオブジェクトは、Azure Active DirectoryのCloud Managerを表し、必要な権限を許可するカスタムロールに割り当てられています。

    次の手順では、新しいAzureポータルを使用します。問題が発生した場合は、Azureクラシックポータルを使用してください。
    ステップ

    1. 必要なCloud Managerの権限を持つカスタムロールを作成します 。

    2. Active Directoryサービスプリンシパルを作成します 。

    3. カスタムCloud Manager Operatorロールをサービスプリンシパルに割り当てます。

    必要なCloud Manager権限を持つカスタムロールを作成する

    カスタムロールは、AzureでCloud Volumes ONTAPを起動および管理するために必要なアクセス許可をCloud Managerに提供するために必要です。

    ステップ

    1. Cloud Manager Azureポリシーをダウンロードしてください。

    2. 割り当て可能なスコープにAzureサブスクリプションIDを追加してJSONファイルを変更します。

      ユーザーがCloud Volumes ONTAPシステムを作成するAzureサブスクリプションごとにIDを追加する必要があります。

      例 

       "AssignableScopes": [ "/subscriptions/d333af45-0d07-4154-943d-c25fbzzzzzzz", "/subscriptions/54b91999-b3e6-4599-908e-416e0zzzzzzz", "/subscriptions/398e471c-3b42-4ae7-9b59-ce5bbzzzzzzz"

    3. JSONファイルを使用してAzureでカスタムロールを作成します。

      次の例は、Azure CLI 2.0を使用してカスタムロールを作成する方法を示しています。

      zロール定義作成 - ロール定義C：\ Policy_for_cloud_Manager_Azure_3.6.1.json

    結果

    これで、OnCommand Cloud Manager Operatorというカスタムロールができました。

    Active Directoryサービスプリンシパルを作成する

    Cloud ManagerがAzure Active Directoryで認証できるように、Active Directoryサービスプリンシパルを作成する必要があります。

    あなたが始める前に

    Active Directoryアプリケーションを作成し、そのアプリケーションをロールに割り当てるには、Azureに適切な権限が必要です。詳細については、 Microsoft Azureのドキュメントを参照してください。ポータルを使用して、リソースにアクセスできるActive Directoryアプリケーションとサービスプリンシパルを作成します 。

    ステップ

    1. Azureポータルから、Azure Active Directoryサービスを開きます。

    2. メニューで、[ App registrations（Legacy） ]をクリックします。

    3. サービスプリンシパルを作成します。

      1. 新規アプリケーション登録をクリックします 。

      2. アプリケーションの名前を入力し、 Webアプリケーション/ APIを選択したままにしてから任意のURLを入力します（例：http：// url）。

      3. 作成をクリックします 。

    4. 必要な権限を追加するようにアプリケーションを変更します。

      1. 作成したアプリケーションを選択してください。

      2. 設定で、 必要なアクセス許可をクリックし、 追加を次にクリックします。

      3. [ APIの選択]をクリックし、[ Windows Azureサービス管理API ]を選択して 、[ 選択 ]をクリックします。

      4. [ 組織ユーザーとしてAzure Service Managementにアクセス ]をクリックし、[ 選択 ]をクリックしてから[ 完了 ]をクリックします。

    5. サービスプリンシパルのキーを作成します。

      1. 設定で、 キーをクリックします。

      2. 説明を入力し、期間を選択して、[ 保存 ]をクリックします 。

      3. キー値をコピーしてください。

        クラウドプロバイダアカウントをCloud Managerに追加するときにキー値を入力する必要があります。

      4. [ プロパティ]をクリックし、サービスプリンシパルのアプリケーションIDをコピーします。

        キー値と同様に、Cloud Managerにクラウドプロバイダーアカウントを追加するときにCloud ManagerにアプリケーションIDを入力する必要があります。

    6. 組織のActive DirectoryテナントIDを取得します。

      1. Active Directoryメニューで、 プロパティをクリックします 。

      2. ディレクトリIDをコピーします。

        アプリケーションIDとアプリケーションキーと同じように、クラウドプロバイダアカウントをCloud Managerに追加するときにActive DirectoryテナントIDを入力する必要があります。

    結果

    これでActive Directoryサービスプリンシパルがあり、アプリケーションID、アプリケーションキー、およびActive DirectoryテナントIDをコピーしたはずです。クラウドプロバイダアカウントを追加するときには、この情報をCloud Managerに入力する必要があります。

    クラウドプリンシパルオペレータロールをサービスプリンシパルに割り当てる

    Cloud ManagerがAzureのアクセス許可を持つように、サービスプリンシパルを1つ以上のAzureサブスクリプションにバインドし、それにCloud Manager Operatorロールを割り当てる必要があります。

    このタスクについて

    複数のAzureサブスクリプションからCloud Volumes ONTAPを展開する場合は、サービスプリンシパルをそれらの各サブスクリプションにバインドする必要があります。 Cloud Managerを使用すると、Cloud Volumes ONTAPをデプロイするときに使用したいサブスクリプションを選択できます。

    ステップ

    1. Azureポータルの左ペインで、[ 購読]を選択します。

    2. 購読を選択してください。

    3. [ アクセス制御（IAM） ] をクリックし、[ 追加 ] をクリックします。

    4. OnCommand Cloud Manager Operatorロールを選択します。

    5. アプリケーションの名前を検索します（スクロールしてもリストに表示されません）。

    6. アプリケーションを選択し、[ 選択 ]、[ OK ]の順にクリックします。

    結果

    Cloud Managerのサービスプリンシパルに、必要なAzure権限が付与されました。

    Cloud ManagerにAzureアカウントを追加する

    必要な権限を持つAzureアカウントを指定したら、そのアカウントをCloud Managerに追加できます。これにより、そのアカウントでCloud Volumes ONTAPシステムを起動できます。

    ステップ

    1. Cloud Managerコンソールの右上にある[設定]アイコンをクリックし、[ クラウドプロバイダとサポートアカウント]を選択します 。

    2. [ 新しいアカウントを追加]をクリックして[ Microsoft Azure ]を選択します 。

    3. 必要な権限を付与するAzure Active Directoryサービスプリンシパルに関する情報を入力します。

    4. ポリシーの要件が満たされていることを確認してから、[ アカウントの作成]をクリックします 。

    結果

    新しい作業環境を作成するときに、Details and Credentialsページから別のアカウントに切り替えることができます。

    追加のAzureサブスクリプションを管理対象IDに関連付ける

    Cloud Managerを使用すると、Cloud Volumes ONTAPを展開するAzureアカウントとサブスクリプションを選択できます。 管理IDをそれらのサブスクリプションに関連付けない限り、管理IDプロファイルに別のAzureサブスクリプションを選択することはできません。

    このタスクについて

    管理IDは、NetApp Cloud CentralからCloud Managerをデプロイしたときの最初のクラウドプロバイダーアカウントです。 Cloud Managerをデプロイしたときに、Cloud CentralはOnCommand Cloud Manager Operatorロールを作成し、それをCloud Manager仮想マシンに割り当てました。

    ステップ

    1. Azureポータルにログインします。

    2. Subscriptionsサービスを開き、Cloud Volumes ONTAPシステムをデプロイするサブスクリプションを選択します。

    3. [ アクセス制御（IAM）]をクリックします。

      1. 「 追加」 >「 役割の割り当てを追加」をクリックしてから、許可を追加します。

        • OnCommand Cloud Manager Operatorロールを選択します。

          OnCommand Cloud Manager Operatorは、 Cloud Managerポリシーで提供されているデフォルトの名前です。役割に別の名前を選択した場合は、代わりにその名前を選択してください。

        • 仮想マシンへのアクセスを割り当てます。

        • Cloud Manager仮想マシンが作成されたサブスクリプションを選択してください。

        • Cloud Manager仮想マシンを選択します。

        • 保存をクリックします 。

    4. 追加のサブスクリプションに対してこれらの手順を繰り返します。

    結果

    新しい作業環境を作成するときに、管理対象のIDプロファイルに対して複数のAzureサブスクリプションから選択できるようになりました。
