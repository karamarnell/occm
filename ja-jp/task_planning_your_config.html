    Cloud Volumes ONTAP構成を計画する GitHubで編集 ドキュメントの変更をリクエストする

    06/14/2019 貢献者

    Cloud Volumes ONTAPをデプロイするときには、ワークロード要件に合った事前構成済みシステムを選択することも、独自の構成を作成することもできます。あなたがあなた自身の設定を選ぶなら、あなたはあなたに利用可能なオプションを理解するべきです。

    ライセンスの種類を選択する

    Cloud Volumes ONTAPは、AWSとAzureの2つの料金オプションで利用できます。2つの料金オプション、つまり「従量課金」と「自分のライセンスを持参」（BYOL）です。従量制では、Explore、Standard、Premiumの3つのライセンスから選択できます。各ライセンスは、異なる容量と計算オプションを提供します。

    • AWSのCloud Volumes ONTAP 9.6でサポートされている構成

    • AzureのCloud Volumes ONTAP 9.6でサポートされている構成

    記憶域の制限について

    Cloud Volumes ONTAPシステムの未処理容量の制限は、ライセンスに関連付けられています。追加の制限は、アグリゲートとボリュームのサイズに影響します。構成を計画する際には、これらの制限に注意する必要があります。

    • AWSでのCloud Volumes ONTAP 9.6のストレージ制限

    • AzureのCloud Volumes ONTAP 9.6のストレージ制限

    AWSでシステムのサイズを設定する

    Cloud VolumesのONTAPシステムをサイジングすると、パフォーマンスと容量の要件を満たすのに役立ちます。インスタンスタイプ、ディスクタイプ、およびディスクサイズを選択する際には、いくつかの重要な点に注意する必要があります。

    インスタンスタイプ

    • ワークロード要件を、各EC2インスタンスタイプの最大スループットとIOPSに合わせます。

    • 複数のユーザーが同時にシステムに書き込む場合は、要求を管理するのに十分なCPUを持つインスタンスタイプを選択してください。

    • あなたがほとんど読んでいるアプリケーションをお持ちの場合は、十分なRAMを持つシステムを選択してください。

      AWSドキュメント：Amazon EC2インスタンスタイプ
      AWSドキュメント：Amazon EBS - 最適化インスタンス

    EBSディスクタイプ

    汎用SSDは、Cloud Volumes ONTAPで最も一般的なディスクタイプです。 EBSディスクの使用例を確認するには、 AWSドキュメント：EBSボリュームタイプを参照してください。

    EBSディスクサイズ

    Cloud Volumes ONTAPシステムを起動するときには、初期ディスクサイズを選択する必要があります。その後、 Cloud Managerにシステムのキャパシティを管理させることができますが、 自分でアグリゲートを構築する場合は、次の点に注意してください 。

    • アグリゲート内のすべてのディスクは同じサイズでなければなりません。

    • EBSディスクのパフォーマンスはディスクサイズに関係しています。サイズによって、SSDディスクのベースラインIOPSと最大バースト期間、およびHDDディスクのベースラインとバーストスループットが決まります。

    • 最終的には、必要な持続パフォーマンスを実現するディスクサイズを選択する必要があります。

    • 大容量のディスクを選択したとしても（たとえば、4 TBディスクが6つ）、EC2インスタンスが帯域幅の制限に達する可能性があるため、すべてのIOPSを取得できない可能性があります。

      EBSディスクのパフォーマンスの詳細については、 AWSのドキュメント：EBSボリュームタイプを参照してください。

    AWSでCloud Volumes ONTAPシステムのサイズを変更する方法の詳細については、以下のビデオをご覧ください。

    Azureでシステムのサイズを変更する

    Cloud VolumesのONTAPシステムをサイジングすると、パフォーマンスと容量の要件を満たすのに役立ちます。 VMタイプ、ディスクタイプ、およびディスクサイズを選択する際には、いくつかの重要な点に注意する必要があります。

    仮想マシンの種類

    『 Cloud Volumes ONTAPリリースノート』でサポートされている仮想マシンの種類を確認してから、サポートされている各VMの種類に関する詳細を確認します。各VMタイプが特定の数のデータディスクをサポートすることに注意してください。

    • Azureのドキュメント：汎用仮想マシンのサイズ

    • Azureドキュメント：メモリ最適化仮想マシンサイズ

    Azureディスクタイプ

    Cloud Volumes ONTAP用のボリュームを作成するときは、Cloud Volumes ONTAPがディスクとして使用する基盤となるクラウドストレージを選択する必要があります。

    HAシステムはプレミアムページBLOBを使用します。一方、シングルノードシステムでは、2種類のAzure Managed Disksを使用できます。

    • Premium SSD管理対象ディスクは、I / Oを多用するワークロードに対して高いコストで高いパフォーマンスを提供します。

    • 標準SSD管理対象ディスクは、低いIOPSを必要とするワークロードに対して一貫したパフォーマンスを提供します。

    • 高いIOPSを必要とせず、コストを削減したい場合は、 標準HDD管理ディスクが適しています。

      これらのディスクの使用例の詳細については、 Microsoft Azureのドキュメント：Microsoft Azureストレージの概要を参照してください。

    Azureディスクサイズ

    Cloud Volumes ONTAPインスタンスを起動するときは、アグリゲートのデフォルトのディスクサイズを選択する必要があります。 Cloud Managerはこのディスクサイズを初期アグリゲートと、単純プロビジョニングオプションを使用したときに作成される追加アグリゲートに使用します。 詳細割り当てオプションを使用して 、デフォルトとは異なるディスクサイズを使用するアグリゲートを作成できます 。

    アグリゲート内のすべてのディスクは同じサイズでなければなりません。

    ディスクサイズを選択するときは、いくつかの要素を考慮する必要があります。ディスクサイズは、ストレージの使用料金、総計で作成できるボリュームのサイズ、Cloud Volumes ONTAPで使用可能な総容量、およびストレージパフォーマンスに影響します。

    Azure Premium Storageのパフォーマンスはディスクサイズと関係があります。ディスクが大きいほど、IOPSとスループットが向上します。たとえば、1 TBのディスクを選択すると、500 GBのディスクよりも高いコストで優れたパフォーマンスが得られます。

    標準記憶域のディスクサイズにパフォーマンスの違いはありません。必要な容量に基づいてディスクサイズを選択する必要があります。

    IOPSとディスクサイズによるスループットについては、Azureを参照してください。

    • Microsoft Azure：マネージドディスクの価格

    • Microsoft Azure：ページブロブ価格

    書き込み速度を選択する

    Cloud Managerを使用すると、単一ノードのCloud Volumes ONTAPシステムの書き込み速度設定を選択できます。書き込み速度を選択する前に、通常の設定と高い設定の違い、および高い書き込み速度を使用する際のリスクと推奨事項について理解しておく必要があります。

    通常の書き込み速度と高速書き込み速度の違い

    通常の書き込み速度を選択すると、データが直接ディスクに書き込まれるため、計画外のシステム停止が発生した場合にデータが失われる可能性が低くなります。

    高速の書き込み速度を選択すると、データはディスクに書き込まれる前にメモリにバッファされ、書き込みパフォーマンスが向上します。このキャッシュにより、計画外のシステム停止が発生した場合にデータが失われる可能性があります。

    計画外のシステム停止が発生した場合に失われる可能性があるデータ量は、最後の2つの整合性ポイントの範囲です。整合点は、バッファされたデータをディスクに書き込むことです。書き込みログがいっぱいになるか、10秒後（どちらか早い方）に整合点が発生します。ただし、AWS EBSボリュームのパフォーマンスは、整合性ポイントの処理時間に影響を与える可能性があります。

    高速書き込みを使用する場合

    ワークロードに高速書き込みパフォーマンスが必要であり、計画外のシステム停止が発生した場合のデータ損失のリスクに耐えることができる場合は、高速書き込み速度が適しています。

    高速書き込みを使用する場合の推奨事項

    高速の書き込み速度を有効にする場合は、アプリケーション層で書き込み保護を確実にする必要があります。

    ボリューム使用量プロファイルの選択

    ONTAPには、必要なストレージの総量を減らすことができるいくつかのストレージ効率化機能が含まれています。 Cloud Managerでボリュームを作成するときは、これらの機能を有効にするプロファイルまたは無効にするプロファイルを選択できます。どのプロファイルを使用するかを決めるのに役立つように、これらの機能についてもっと学ぶべきです。

    NetAppのストレージ効率化機能には、次のような利点があります。

    シンプロビジョニング

    実際に物理ストレージプールにあるよりも多くの論理ストレージをホストまたはユーザーに提示します。記憶スペースを事前に割り当てるのではなく、データが書き込まれるときに記憶スペースが各ボリュームに動的に割り当てられます。

    重複排除

    同一のデータブロックを探し出し、それらを単一の共有ブロックへの参照に置き換えることによって効率を向上させます。この手法では、同じボリュームに存在するデータの冗長ブロックを排除することで、記憶容量の要件を軽減します。

    圧縮

    1次ストレージ、2次ストレージ、およびアーカイブ・ストレージ上のボリューム内のデータを圧縮することによって、データの保管に必要な物理容量を削減します。
