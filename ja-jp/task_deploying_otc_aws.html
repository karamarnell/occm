    AWSでクラウドボリュームONTAPを起動する GitHubで編集 ドキュメントの変更をリクエストする

    05/01/2019 貢献者

    Cloud Volumes ONTAPは、単一システム構成でも、AWSのHAペアとしても起動できます。

    AWSで単一のCloud Volumes ONTAPシステムを起動する

    AWSでCloud Volumes ONTAPを起動する場合は、Cloud Managerで新しい作業環境を作成する必要があります。

    あなたが始める前に

    • 設定を選択し、管理者からAWSネットワーキング情報を取得することによって準備しておく必要があります。詳しくは、 Cloud Volumes ONTAP構成の計画を参照してください。

    • BYOLシステムを起動したい場合は、20桁のシリアル番号（ライセンスキー）が必要です。

    • CIFSを使用したい場合は、DNSとActive Directoryを設定しておく必要があります。詳細については、 AWSのCloud Volumes ONTAPのネットワーク要件を参照してください。

    このタスクについて

    作業環境を作成した直後に、Cloud Managerは指定されたVPCでテストインスタンスを起動して接続を確認します。成功すると、Cloud Managerはすぐにインスタンスを終了してからCloud Volumes ONTAPシステムのデプロイを開始します。 Cloud Managerが接続を確認できない場合、作業環境の作成は失敗します。テストインスタンスは、t2.nano（デフォルトVPCテナントの場合）またはm3.medium（専用VPCテナントの場合）です。

    ステップ

    1. 「作業環境」ページで、「作業環境の追加 」をクリックします。

    2. [作成]の下で、[ Cloud Volumes ONTAP]を選択します。

    3. [詳細と認証情報]ページで、オプションでAWSアカウントを変更し、作業環境名を入力し、必要に応じてタグを追加してから、パスワードを入力します。

      このページのいくつかのフィールドは一目瞭然です。次の表は、ガイダンスが必要となる可能性があるフィールドについて説明しています。

      フィールド説明

      アカウントを切り替える

      追加のクラウドプロバイダアカウントを追加した場合は、別のアカウントを選択できます。詳細については、 Cloud Managerへのクラウドプロバイダーアカウントの追加を参照してください。

      作業環境名

      Cloud Managerは、作業環境名を使用してCloud Volumes ONTAPシステムとAmazon EC2インスタンスの両方に名前を付けます。このオプションを選択した場合は、定義済みセキュリティグループの接頭辞としても名前が使用されます。

      タグを追加する

      AWSタグは、AWSリソースのメタデータです。 Cloud Managerは、Cloud Volumes ONTAPインスタンスとそのインスタンスに関連付けられている各AWSリソースにタグを追加します。

      作業環境を作成するときにユーザーインターフェイスから最大4つのタグを追加でき、作成後にさらに追加できます。作業環境を作成するとき、APIはあなたを4つのタグに制限しないことに注意してください。

      タグについては、 AWSのドキュメント：Amazon EC2リソースへのタグ付けを参照してください 。

      資格情報

      これらはCloud Volumes ONTAPクラスター管理者アカウントの資格情報です。これらの認証情報を使用して、OnCommand System ManagerまたはそのCLIを介してCloud Volumes ONTAPに接続できます。

      Cloud ManagerアカウントにAWSキーが指定されていない場合は、[続行]をクリックした後にそれらを入力するように求められます。先に進む前にそれらを入力する必要があります。

    4. [Location＆Connectivity]ページで、AWSワークシートに記録したネットワーク情報を入力して[ Continue ]をクリックします。

      次の図は、[Location＆Connectivity]ページに入力されているところを示しています。

    5. [データの暗号化]ページで、[データの暗号化なし]または[AWS管理の暗号化]を選択します。

      AWS管理の暗号化では、自分のアカウントまたは別のAWSアカウントとは異なるカスタマーマスターキー（CMK）を選択できます。

      Cloud Volumes ONTAP用のAWS KMSをセットアップする方法を学びます 。

      サポートされている暗号化テクノロジの詳細

    6. [ライセンスとサポートサイトアカウント]ページで、[従量課金]または[BYOL]のどちらを使用するかを指定してから、NetAppサポートサイトアカウントを指定します。

      ライセンスの仕組みを理解するには、参照のライセンスを 。

      NetAppサポートサイトアカウントは、従量制ではオプションですが、BYOLシステムでは必須です。 NetAppサポートサイトアカウントを追加する方法を学びます 。

    7. [Preconfigured Packages]ページで、Cloud Volumes ONTAPをすばやく起動するためのパッケージを1つ選択するか、[ Create my own configuration ]をクリックします 。

      いずれかのパッケージを選択した場合は、ボリュームを指定してから構成を確認して承認するだけで済みます。

    8. [IAMロール]ページでは、Cloud Managerにロールを作成させるデフォルトのオプションをそのまま使用する必要があります。

      独自のポリシーを使用したい場合は、Cloud Volumes ONTAPノードのポリシー要件を満たす必要があります 。

    9. [ライセンス]ページで、必要に応じてCloud Volumes ONTAPのバージョンを変更し、ライセンス、インスタンスタイプ、インスタンステナントを選択し、[ 続行 ]をクリックします。

      インスタンスを起動した後にニーズが変わった場合は、後でライセンスまたはインスタンスタイプを変更できます。

      選択したバージョンで新しいリリース候補版、一般出荷版、またはパッチリリースが利用可能な場合、Cloud Managerは作業環境を作成するときにシステムをそのバージョンに更新します。たとえば、Cloud Volumes ONTAP 9.4 RC1および9.4 GAが利用可能であると選択した場合、更新が行われます。更新はリリース間では発生しません。たとえば、9.3から9.4までです。

    10. [Underlying Storage Resources]ページで、初期アグリゲートの設定（ディスクタイプ、各ディスクのサイズ、およびS3階層化を有効にするかどうか）を選択します。

      ディスクタイプは初期ボリューム用です。後続のボリュームには異なるディスクタイプを選択できます。

      ディスクサイズは、初期アグリゲート内のすべてのディスク、およびシンプルプロビジョニングオプションを使用したときにCloud Managerによって作成される追加アグリゲート用です。詳細割り当てオプションを使用して、異なるディスクサイズを使用するアグリゲートを作成できます。

      ディスクの種類とサイズの選択については、AWSでシステムのサイズ設定を参照してください。

    11. 必要に応じて、書き込み速度＆WORMページで、 ノーマルまたは高速書き込み速度を選択し、活性化させるには、多くの（WORM）ストレージを読んで、一度書き込み。

      書き込み速度について詳しく学んでください 。

      WORMストレージについてさらに学んでください 。

    12. [ボリュームの作成]ページで、新しいボリュームの詳細を入力し、[ 続行 ]をクリックします。

      iSCSI用のボリュームを作成する場合は、この手順を省略することができます。 Cloud ManagerはNFSとCIFS専用のボリュームを設定します。

      このページのいくつかのフィールドは一目瞭然です。次の表は、ガイダンスが必要となる可能性があるフィールドについて説明しています。

      フィールド説明

      サイズ

      入力できる最大サイズは、シンプロビジョニングを有効にしているかどうかによって大きく異なります。これにより、現在利用可能な物理ストレージよりも大きいボリュームを作成できます。

      アクセス制御（NFSのみ）

      エクスポートポリシーは、ボリュームにアクセスできるサブネット内のクライアントを定義します。デフォルトでは、Cloud Managerはサブネット内のすべてのインスタンスへのアクセスを提供する値を入力します。

      権限とユーザー/グループ（CIFSのみ）

      これらのフィールドを使用すると、ユーザーおよびグループの共有へのアクセスレベル（アクセス制御リストまたはACLとも呼ばれます）を制御できます。ローカルまたはドメインのWindowsユーザーまたはグループ、あるいはUNIXユーザーまたはグループを指定できます。ドメインのWindowsユーザー名を指定する場合は、domain \ usernameの形式でユーザーのドメインを含める必要があります。

      スナップショットポリシー

      Snapshotコピーポリシーは、自動的に作成されたNetApp Snapshotコピーの頻度と数を指定します。 NetApp Snapshotコピーは、パフォーマンスへの影響がなく、最小限の記憶域しか必要としないポイントインタイムファイルシステムイメージです。デフォルトのポリシーを選択することも、noneを選択することもできます。一時データにはnoneを選択することができます。たとえば、Microsoft SQL Serverの場合はtempdbです。

      次の図は、CIFSプロトコル用に入力された[Volume]ページを示しています。

    13. CIFSプロトコルを選択した場合は、[CIFS Setup]ページでCIFSサーバを設定します。

      フィールド説明

      DNSのプライマリおよびセカンダリIPアドレス

      CIFSサーバの名前解決を提供するDNSサーバのIPアドレス。
      表示されたDNSサーバーには、CIFSサーバーが参加するドメインのActive Directory LDAPサーバーとドメインコントローラーを見つけるために必要なサービスロケーションレコード（SRV）が含まれている必要があります。

      参加するActive Directoryドメイン

      CIFSサーバを参加させたいActive Directory（AD）ドメインのFQDN。

      ドメインに参加することを承認された資格情報

      ADドメイン内の指定された組織単位（OU）にコンピューターを追加するのに十分な特権を持つWindowsアカウントの名前とパスワード。

      CIFSサーバのNetBIOS名

      ADドメイン内で一意のCIFSサーバ名。

      組織ユニット

      ADドメイン内のCIFSサーバーに関連付ける組織単位。デフォルトはCN = Computersです。

      DNSドメイン

      Cloud Volumes ONTAPストレージ仮想マシン（SVM）のDNSドメイン。ほとんどの場合、ドメインはADドメインと同じです。

      NTPサーバ

      Active Directory DNSを使用してNTPサーバーを構成するには、 Active Directoryドメインを使用を選択します。別のアドレスを使用してNTPサーバーを構成する必要がある場合は、APIを使用してください。詳細については、 Cloud Manager API開発者ガイドを参照してください。

    14. [Usage Profile]、[Disk Type]、および[Tiering Policy]ページで、必要に応じて、ストレージ効率化機能を有効にしてS3階層化ポリシーを編集するかどうかを選択します。

      詳細については、 「ボリューム使用状況プロファイルについて」および「 データ階層化の概要 」を参照してください。

    15. [Review＆Approve]ページで、選択内容を確認して確認します。

      1. 構成に関する詳細を確認してください。

      2. Cloud Managerが購入するサポートとAWSリソースの詳細を確認するには、[ More information]をクリックします。

      3. チェックボックスをオン...私は理解して選択します。

      4. Goをクリックしてください。

    結果

    Cloud ManagerがCloud Volumes ONTAPインスタンスを起動します。タイムラインで進行状況を追跡できます。

    Cloud Volumes ONTAPインスタンスの起動で問題が発生した場合は、失敗のメッセージを確認してください。作業環境を選択して「環境の再作成」をクリックすることもできます。

    追加の支援が必要な場合は、 NetApp Cloud Volumes ONTAP Supportにアクセスしてください。

    終了したら

    • CIFS共有をプロビジョニングした場合は、ユーザーまたはグループにファイルとフォルダへのアクセス許可を付与し、それらのユーザーが共有にアクセスしてファイルを作成できることを確認します。

    • クォータをボリュームに適用する場合は、システムマネージャまたはCLIを使用してください。

      割り当て制限を使用すると、ユーザー、グループ、またはqtreeが使用するディスク容量とファイル数を制限または追跡できます。

    AWSでCloud Volumes ONTAP HAペアを起動する

    AWSでCloud Volumes ONTAP HAペアを起動したい場合は、Cloud ManagerでHA作業環境を作成する必要があります。

    • CIFSを使用したい場合は、DNSとActive Directoryを設定しておく必要があります。詳細については、 AWSのCloud Volumes ONTAPのネットワーク要件を参照してください。

    3. [詳細と認証情報]ページで、オプションでAWSアカウントを変更し、作業環境名を入力し、必要に応じてタグを追加してから、パスワードを入力します。

    4. HA配置モデルページで、HA設定を選択します。

      展開モデルの概要については、 Cloud Volumes ONTAP HA for AWSを参照してください。

    5. [Region＆VPC]ページで、AWSワークシートに記録したネットワーク情報を入力して[ Continue ]をクリックします。

      次の図は、複数のAZ構成に対して入力された[場所]ページを示しています。

    6. [接続とSSH認証]ページで、HAペアとメディエータの接続方法を選択します。

    7. 複数のAZを選択した場合は、浮動IPアドレスを指定して[ 続行 ]をクリックします。

      領域内のすべてのVPCについて、IPアドレスはCIDRブロックの外側になければなりません。詳細については、 複数のAZでのCloud Volumes ONTAP HAのAWSネットワーク要件を参照してください。

    8. 複数のAZを選択した場合は、フローティングIPアドレスへのルートを含める必要があるルートテーブルを選択し、[ 続行 ]をクリックします。

      複数のルートテーブルがある場合は、正しいルートテーブルを選択することが非常に重要です。そうでないと、一部のクライアントがCloud Volumes ONTAP HAペアにアクセスできない可能性があります。ルートテーブルの詳細については、 AWSドキュメント：Route Tablesを参照してください。

    9. [データの暗号化]ページで、[データの暗号化なし]または[AWS管理の暗号化]を選択します。

      AWS管理の暗号化では、自分のアカウントまたは別のAWSアカウントとは異なるカスタマーマスターキー（CMK）を選択できます。

      Cloud Volumes ONTAP用のAWS KMSをセットアップする方法を学びます 。

      サポートされている暗号化テクノロジの詳細

    10. [ライセンスとサポートサイトアカウント]ページで、[従量課金]または[BYOL]のどちらを使用するかを指定してから、NetAppサポートサイトアカウントを指定します。

      ライセンスの仕組みを理解するには、参照のライセンスを 。

      NetAppサポートサイトアカウントは、従量制ではオプションですが、BYOLシステムでは必須です。 NetAppサポートサイトアカウントを追加する方法を学びます 。

    11. [Preconfigured Packages]ページで、Cloud Volumes ONTAPシステムをすばやく起動するためのパッケージを1つ選択するか、[ Create my own configuration ]をクリックします 。

      いずれかのパッケージを選択した場合は、ボリュームを指定してから構成を確認して承認するだけで済みます。

    12. [IAMロール]ページでは、Cloud Managerにロールを作成させるデフォルトのオプションをそのまま使用する必要があります。

      独自のポリシーを使用する場合は、Cloud Volumes ONTAPノードとHAメディエータのポリシー要件を満たす必要があります 。

    13. [ライセンス]ページで、必要に応じてCloud Volumes ONTAPのバージョンを変更し、ライセンス、インスタンスタイプ、インスタンステナントを選択し、[ 続行 ]をクリックします。

      インスタンスを起動した後にニーズが変わった場合は、後でライセンスまたはインスタンスタイプを変更できます。

      選択したバージョンで新しいリリース候補版、一般出荷版、またはパッチリリースが利用可能な場合、Cloud Managerは作業環境を作成するときにシステムをそのバージョンに更新します。たとえば、Cloud Volumes ONTAP 9.4 RC1および9.4 GAが利用可能であると選択した場合、更新が行われます。更新はリリース間では発生しません。たとえば、9.3から9.4までです。

    14. [Underlying Storage Resources]ページで、初期アグリゲートの設定（ディスクタイプ、各ディスクのサイズ、およびS3階層化を有効にするかどうか）を選択します。

      ディスクタイプは初期ボリューム用です。後続のボリュームには異なるディスクタイプを選択できます。

      ディスクサイズは、初期アグリゲート内のすべてのディスク、およびシンプルプロビジョニングオプションを使用したときにCloud Managerによって作成される追加アグリゲート用です。詳細割り当てオプションを使用して、異なるディスクサイズを使用するアグリゲートを作成できます。

      ディスクの種類とサイズの選択については、AWSでシステムのサイズ設定を参照してください。

    15. 必要に応じて、「WORM」ページで、1回書き込み、多数（WORM）ストレージを読み取ります。

      WORMストレージについてさらに学んでください 。

    16. [ボリュームの作成]ページで、新しいボリュームの詳細を入力し、[ 続行 ]をクリックします。

      iSCSI用のボリュームを作成する場合は、この手順を省略することができます。 Cloud ManagerはNFSとCIFS専用のボリュームを設定します。

      このページのいくつかのフィールドは一目瞭然です。次の表は、ガイダンスが必要となる可能性があるフィールドについて説明しています。

      フィールド説明

      サイズ

      入力できる最大サイズは、シンプロビジョニングを有効にしているかどうかによって大きく異なります。これにより、現在利用可能な物理ストレージよりも大きいボリュームを作成できます。

      アクセス制御（NFSのみ）

      エクスポートポリシーは、ボリュームにアクセスできるサブネット内のクライアントを定義します。デフォルトでは、Cloud Managerはサブネット内のすべてのインスタンスへのアクセスを提供する値を入力します。

      権限とユーザー/グループ（CIFSのみ）

      これらのフィールドを使用すると、ユーザーおよびグループの共有へのアクセスレベル（アクセス制御リストまたはACLとも呼ばれます）を制御できます。ローカルまたはドメインのWindowsユーザーまたはグループ、あるいはUNIXユーザーまたはグループを指定できます。ドメインのWindowsユーザー名を指定する場合は、domain \ usernameの形式でユーザーのドメインを含める必要があります。

      スナップショットポリシー

      Snapshotコピーポリシーは、自動的に作成されたNetApp Snapshotコピーの頻度と数を指定します。 NetApp Snapshotコピーは、パフォーマンスへの影響がなく、最小限の記憶域しか必要としないポイントインタイムファイルシステムイメージです。デフォルトのポリシーを選択することも、noneを選択することもできます。一時データにはnoneを選択することができます。たとえば、Microsoft SQL Serverの場合はtempdbです。

      次の図は、CIFSプロトコル用に入力された[Volume]ページを示しています。

    17. CIFSプロトコルを選択した場合は、[CIFS設定]ページでCIFSサーバを設定します。

      フィールド説明

      DNSのプライマリおよびセカンダリIPアドレス

      CIFSサーバの名前解決を提供するDNSサーバのIPアドレス。
      表示されたDNSサーバーには、CIFSサーバーが参加するドメインのActive Directory LDAPサーバーとドメインコントローラーを見つけるために必要なサービスロケーションレコード（SRV）が含まれている必要があります。

      参加するActive Directoryドメイン

      CIFSサーバを参加させたいActive Directory（AD）ドメインのFQDN。

      ドメインに参加することを承認された資格情報

      ADドメイン内の指定された組織単位（OU）にコンピューターを追加するのに十分な特権を持つWindowsアカウントの名前とパスワード。

      CIFSサーバのNetBIOS名

      ADドメイン内で一意のCIFSサーバ名。

      組織ユニット

      ADドメイン内のCIFSサーバーに関連付ける組織単位。デフォルトはCN = Computersです。

      DNSドメイン

      Cloud Volumes ONTAPストレージ仮想マシン（SVM）のDNSドメイン。ほとんどの場合、ドメインはADドメインと同じです。

      NTPサーバ

      Active Directory DNSを使用してNTPサーバーを構成するには、 Active Directoryドメインを使用を選択します。別のアドレスを使用してNTPサーバーを構成する必要がある場合は、APIを使用してください。詳細については、 Cloud Manager API開発者ガイドを参照してください。

    18. [Usage Profile]、[Disk Type]、および[Tiering Policy]ページで、必要に応じて、ストレージ効率化機能を有効にしてS3階層化ポリシーを編集するかどうかを選択します。

      詳細については、 「ボリューム使用状況プロファイルについて」および「 データ階層化の概要 」を参照してください。

    19. [Review＆Approve]ページで、選択内容を確認して確認します。

      1. 構成に関する詳細を確認してください。

      2. Cloud Managerが購入するサポートとAWSリソースの詳細を確認するには、[ More information]をクリックします。

      3. チェックボックスをオン...私は理解して選択します。

      4. Goをクリックしてください。

    結果

    Cloud ManagerがCloud Volumes ONTAP HAペアを起動します。タイムラインで進行状況を追跡できます。

    HAペアの起動に問題が発生した場合は、失敗のメッセージを確認してください。作業環境を選択して「環境の再作成」をクリックすることもできます。

    追加の支援が必要な場合は、 NetApp Cloud Volumes ONTAP Supportにアクセスしてください。

    終了したら

    • CIFS共有をプロビジョニングした場合は、ユーザーまたはグループにファイルとフォルダへのアクセス許可を付与し、それらのユーザーが共有にアクセスしてファイルを作成できることを確認します。

    • クォータをボリュームに適用する場合は、システムマネージャまたはCLIを使用してください。

      割り当て制限を使用すると、ユーザー、グループ、またはqtreeが使用するディスク容量とファイル数を制限または追跡できます。
