    Cloud Managerのネットワーク要件 GitHubで編集 ドキュメントの変更をリクエストする

    06/12/2019 貢献者

    Cloud ManagerがCloud Volumes ONTAPシステムをAWSまたはMicrosoft Azureにデプロイできるように、ネットワークを設定する必要があります。最も重要なステップは、さまざまなエンドポイントへのアウトバウンドインターネットアクセスを確実にすることです。

    ネットワークがインターネットへのすべての通信にプロキシサーバーを使用している場合、Cloud Managerはセットアップ中にプロキシを指定するように要求します。設定ページからプロキシサーバーを指定することもできます。 プロキシサーバーを使用するようにCloud Managerを設定するを参照してください。

    ターゲットネットワークへの接続

    Cloud Managerには、Cloud Volumes ONTAPをデプロイするAWS VPCとAzure VNetへのネットワーク接続が必要です。

    たとえば、Cloud Managerを企業ネットワークにインストールする場合は、Cloud Volumes ONTAPを起動するAWS VPCまたはAzure VNetへのVPN接続を設定する必要があります。

    インターネットへのアクセス

    Cloud Managerは、Cloud Volumes ONTAPをデプロイおよび管理するためにアウトバウンドインターネットアクセスを必要とします。 WebブラウザからCloud Managerにアクセスするとき、およびLinuxホストでCloud Managerインストーラを実行するときにも、アウトバウンドインターネットアクセスが必要です。

    以下のセクションでは、特定のエンドポイントについて説明します。

    AWSでCloud Volumes ONTAPを管理するためのアウトバウンドインターネットアクセス

    Cloud Managerは、AWSでCloud Volumes ONTAPをデプロイおよび管理するときに、次のエンドポイントに連絡するためのアウトバウンドインターネットアクセスを必要とします。

    エンドポイント目的

    AWSサービス（amazonaws.com）：

    • CloudFormation

    • Elastic Compute Cloud（EC2）

    • 鍵管理サービス（KMS）

    • セキュリティトークンサービス（STS）

    • 簡易保管サービス（S3）

    正確なエンドポイントは、Cloud Volumes ONTAPをデプロイする地域によって異なります。 詳細についてはAWSのドキュメントを参照してください。

    Cloud ManagerがAWSでCloud Volumes ONTAPをデプロイおよび管理できるようにします。

    https://api.services.cloud.netapp.com:443

    NetApp Cloud CentralへのAPIリクエスト。

    https://cloud.support.netapp.com.s3.us-west-1.amazonaws.com

    ソフトウェアイメージ、マニフェスト、およびテンプレートへのアクセスを提供します。

    https://cognito-idp.us-east-1.amazonaws.com
    https://cognito-identity.us-east-1.amazonaws.com

    Cloud Managerがマニフェスト、テンプレート、およびCloud Volumes ONTAPアップグレードイメージにアクセスしてダウンロードできるようにします。

    https://kinesis.us-east-1.amazonaws.com

    NetAppが監査レコードからデータをストリーミングできるようにします。

    https://cloudmanager.cloud.netapp.com

    Cloud Centralアカウントを含むCloud Managerサービスとの通信。

    Webブラウザからインターネットへのアクセス

    ユーザーはWebブラウザからCloud Managerにアクセスする必要があります。 Webブラウザを実行しているマシンは、次のエンドポイントに接続している必要があります。

    エンドポイント目的

    Cloud Managerホスト

    Cloud Managerコンソールをロードするには、WebブラウザからホストのIPアドレスを入力する必要があります。

    クラウドプロバイダへの接続性に応じて、ホストに割り当てられているプライベートIPまたはパブリックIPを使用できます。

    • VPNがあり、仮想ネットワークへの直接接続アクセスがある場合は、プライベートIPが機能します。

    • パブリックIPはあらゆるネットワークシナリオで機能します。

    いずれにせよ、セキュリティグループのルールで、許可されたIPまたはサブネットからのアクセスのみを許可するようにして、ネットワークアクセスを保護する必要があります。

    https://auth0.com
    https://cdn.auth0.com
    https://netapp-cloud-account.auth0.com
    https://services.cloud.netapp.com

    Webブラウザはこれらのエンドポイントに接続し、NetApp Cloud Centralを介した集中ユーザー認証を行います。

    https://widget.intercom.io

    NetAppクラウドの専門家に相談することを可能にする製品内チャット用。

    Cloud Managerインストーラーは、インストールプロセス中に以下のURLにアクセスする必要があります。

    • http://dev.mysql.com/get/mysql-community-release-el7-5.noarch.rpm

    • https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm

    • https://s3.amazonaws.com/aws-cli/awscli-bundle.zip

    • 既存のLinuxホストにCloud Managerをインストールする場合は、 Cloud Managerホストの要件を参照してください。
