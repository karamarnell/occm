    Cloud Volumes ONTAPソフトウェアの更新 GitHubで編集 ドキュメントの変更をリクエストする

    09/10/2019 寄稿者

    Cloud Managerには、現在のCloud Volumes ONTAPリリースにアップグレードしたり、Cloud Volumes ONTAPを以前のリリースにダウングレードしたりするために使用できるいくつかのオプションが含まれています。ソフトウェアをアップグレードまたはダウングレードする前に、Cloud Volumes ONTAPシステムを準備する必要があります。

    概要

    Cloud Managerは、新しいバージョンのCloud Volumes ONTAPが利用可能になると、Cloud Volumes ONTAP作業環境に通知を表示します。

    この通知からアップグレードプロセスを開始できます。このプロセスは、S3バケットからソフトウェアイメージを取得し、イメージをインストールしてからシステムを再起動することでプロセスを自動化します。詳細については、 Cloud Volumes ONTAPの最新バージョンへのアップグレードを参照してください。

    HAシステムの場合、Cloud Managerはアップグレードプロセスの一部としてHAメディエーターをアップグレードする場合があります。

    ソフトウェア更新の詳細オプション

    Cloud Managerは、Cloud Volumes ONTAPソフトウェアを更新するための次の高度なオプションも提供します。

    • 外部URLの画像を使用したソフトウェアの更新

      このオプションは、Cloud ManagerがソフトウェアをアップグレードするためにS3バケットにアクセスできない場合、パッチが提供された場合、またはソフトウェアを特定のバージョンにダウングレードする場合に役立ちます。

      詳細については、 HTTPまたはFTPサーバーを使用したCloud Volumes ONTAPのアップグレードまたはダウングレードを参照してください。

    • システム上の代替イメージを使用したソフトウェアの更新

      このオプションを使用して、代替ソフトウェアイメージをデフォルトイメージにすることにより、以前のバージョンにダウングレードできます。このオプションは、HAペアでは使用できません。

      詳細については、 ローカルイメージを使用したCloud Volumes ONTAPのダウングレードを参照してください。

    Cloud Volumes ONTAPソフトウェアの更新準備

    アップグレードまたはダウングレードを実行する前に、システムの準備ができていることを確認し、必要な構成変更を行う必要があります。

    • ダウンタイムの計画

    • バージョン要件の確認

    • SnapMirror転送の一時停止

    • アグリゲートがオンラインであることを確認する

    ダウンタイムの計画

    単一ノードシステムをアップグレードすると、アップグレードプロセスによりシステムが最大25分間オフラインになり、その間I / Oが中断されます。

    HAペアのアップグレードは無停止です。無停止アップグレードは、クライアントへのサービスを維持しながら、HAペアの両方のノードを同時にアップグレードします。

    バージョン要件の確認

    アップグレードまたはダウングレードできるONTAPのバージョンは、システムで現在実行されているONTAPのバージョンによって異なります。

    バージョンの要件を理解するには、 ONTAP 9のドキュメント：クラスターの更新要件を参照してください。

    SnapMirror転送の一時停止

    Cloud Volumes ONTAPシステムにアクティブなSnapMirror関係がある場合、Cloud Volumes ONTAPソフトウェアを更新する前に転送を一時停止することをお勧めします。転送を一時停止すると、SnapMirrorの障害が防止されます。宛先システムからの転送を一時停止する必要があります。

    このタスクについて

    これらの手順では、バージョン9.3以降のSystem Managerの使用方法について説明します。

    手順

    1. 宛先システムからSystem Managerにログインします。

    2. [ 保護]> [関係]をクリックします。

    3. 関係を選択し、「 操作」>「静止」をクリックします。

    アグリゲートがオンラインであることを確認する

    Cloud Volumes ONTAPのアグリゲートは、ソフトウェアを更新する前にオンラインにする必要があります。集計はほとんどの構成でオンラインにする必要がありますが、そうでない場合は、オンラインにする必要があります。

    このタスクについて

    これらの手順では、バージョン9.3以降のSystem Managerの使用方法について説明します。

    手順

    1. 作業環境で、メニューアイコンをクリックし、[ 詳細 ] > [ 高度な割り当て ]をクリックします。

    2. アグリゲートを選択し、[ 情報 ]をクリックして、状態がオンラインであることを確認します。

    3. アグリゲートがオフラインの場合、System Managerを使用してアグリゲートをオンラインにします。

      1. System Managerにログインします。

      2. [ ストレージ]> [集計とディスク]> [集計]をクリックします。

      3. 集約を選択し、[ その他のアクション ] > [ステータス]> [オンライン ]をクリックします。

    Cloud Volumes ONTAPを最新バージョンにアップグレードする

    Cloud Volumes ONTAPの最新バージョンにCloud Managerから直接アップグレードできます。 Cloud Managerは、新しいバージョンが利用可能になると通知します。

    あなたが始める前に

    Cloud Volumes ONTAPシステムでは、ボリュームやアグリゲートの作成などのCloud Manager操作が進行中でない必要があります。

    このタスクについて

    • 単一ノードシステムをアップグレードすると、アップグレードプロセスによりシステムが最大25分間オフラインになり、その間I / Oが中断されます。

    • HAペアのアップグレードは無停止です。無停止アップグレードは、クライアントへのサービスを維持しながら、HAペアの両方のノードを同時にアップグレードします。

    手順

    1. 作業環境をクリックします。

    2. 作業環境を選択します。

      新しいバージョンが利用可能な場合、右側のペインに通知が表示されます。

    3. 新しいバージョンが利用可能な場合は、[ アップグレード ]をクリックします。

    4. リリース情報ページでは、指定されたバージョンのリリースノートを読むためにリンクをクリックし、ボックスをチェック...私は読んだことを選択します。

    5. エンドユーザーライセンス契約（EULA）ページでEULA を読み、 [EULA を読んで承認する]を選択します 。

    6. ページレビューではと承認、重要な注意事項を読んで、 私は理解して...]を選択し、[Go]をクリックします。

    結果

    Cloud Managerがソフトウェアのアップグレードを開始します。ソフトウェアの更新が完了すると、作業環境でアクションを実行できます。

    終わったら

    SnapMirror転送を中断した場合は、System Managerを使用して転送を再開します。

    HTTPまたはFTPサーバーを使用したCloud Volumes ONTAPのアップグレードまたはダウングレード

    Cloud Volumes ONTAPソフトウェアイメージをHTTPまたはFTPサーバーに配置してから、Cloud Managerからソフトウェアの更新を開始できます。 Cloud ManagerがS3バケットにアクセスしてソフトウェアをアップグレードできない場合、またはソフトウェアをダウングレードする場合、このオプションを使用できます。

    このタスクについて

    • 単一ノードシステムをアップグレードすると、アップグレードプロセスによりシステムが最大25分間オフラインになり、その間I / Oが中断されます。

    • HAペアのアップグレードは無停止です。無停止アップグレードは、クライアントへのサービスを維持しながら、HAペアの両方のノードを同時にアップグレードします。

    手順

    1. Cloud Volumes ONTAPソフトウェアイメージをホストできるHTTPサーバーまたはFTPサーバーをセットアップします。

    2. 仮想ネットワークへのVPN接続がある場合、Cloud Volumes ONTAPソフトウェアイメージを独自のネットワークのHTTPサーバーまたはFTPサーバーに配置できます。それ以外の場合は、クラウド内のHTTPサーバーまたはFTPサーバーにファイルを配置する必要があります。

    3. Cloud Volumes ONTAPに独自のセキュリティグループを使用する場合は、Cloud Volumes ONTAPがソフトウェアイメージにアクセスできるように、アウトバウンドルールがHTTPまたはFTP接続を許可するようにしてください。

      定義済みのCloud Volumes ONTAPセキュリティグループは、デフォルトでアウトバウンドHTTPおよびFTP接続を許可します。

    4. ネットアップサポートサイトからソフトウェアイメージを取得します。

    5. ファイルの提供元となるHTTPまたはFTPサーバーのディレクトリにソフトウェアイメージをコピーします。

    6. Cloud Managerの作業環境で、メニューアイコンをクリックし、[ 詳細 ] > [ Cloud Volumes ONTAPの更新 ]をクリックします。

    7. ソフトウェアの更新ページで、 [URLから利用可能な画像を選択]を選択し、URLを入力して、[ 画像の変更 ]をクリックします。

    8. [ 続行]をクリックして確認します。

    結果

    Cloud Managerがソフトウェアの更新を開始します。ソフトウェアの更新が完了すると、作業環境でアクションを実行できます。

    終わったら

    SnapMirror転送を中断した場合は、System Managerを使用して転送を再開します。

    ローカルイメージを使用したCloud Volumes ONTAPのダウングレード

    Cloud Volumes ONTAPを同じリリースファミリーの以前のリリース（9.5から9.4など）に移行することをダウングレードと呼びます。新しいクラスターまたはテストクラスターをダウングレードする場合、支援なしでダウングレードできますが、運用クラスターをダウングレードする場合は、テクニカルサポートにお問い合わせください。

    各Cloud Volumes ONTAPシステムは、実行中の現在のイメージと起動可能な代替イメージの2つのソフトウェアイメージを保持できます。 Cloud Managerは、代替イメージをデフォルトイメージに変更できます。現在のイメージで問題が発生している場合、このオプションを使用して、Cloud Volumes ONTAPの以前のバージョンにダウングレードできます。

    このタスクについて

    このダウングレードプロセスは、単一のCloud Volumes ONTAPシステムでのみ使用できます。 HAペアでは使用できません。このプロセスにより、Cloud Volumes ONTAPシステムが最大25分間オフラインになります。

    手順

    1. 作業環境から、メニューアイコンをクリックし、[ 詳細 ] > [ Cloud Volumes ONTAPの更新 ]をクリックします。

    2. ソフトウェアの更新ページで、代替イメージを選択し、[ イメージの変更 ]をクリックします。

    3. [ 続行]をクリックして確認します。

    結果

    Cloud Managerがソフトウェアの更新を開始します。ソフトウェアの更新が完了すると、作業環境でアクションを実行できます。

    終わったら

    SnapMirror転送を中断した場合は、System Managerを使用して転送を再開します。
