    Cloud Manager 3.7の新機能 GitHubで編集 ドキュメントの変更をリクエストする

    12/10/2019 寄稿者

    RSS購読

    Cloud Managerは通常、毎月新しいリリースを導入して、新しい機能、拡張機能、およびバグ修正を提供します。

    以前のリリースをお探しですか？
    3.6の新機能
    3.5の新機能
    3.4の新機能

    Cloud Manager 3.7.5（2019年12月3日）

    Cloud Manager 3.7.5には、次の機能強化が含まれています。

    • GCPのCloud Volumes ONTAPの高速書き込み速度

    • Kubernetesの永続ストレージとしてのオンプレミスONTAPクラスター

    • Kubernetesの最新のトライデントバージョン

    • Azure汎用v2ストレージアカウントのサポート

    • APIを使用したAzureストレージアカウント名のプレフィックス

    GCPのCloud Volumes ONTAPの高速書き込み速度

    Google Cloud Platformの新規および既存のCloud Volumes ONTAPシステムで高速の書き込み速度を有効にできるようになりました。ワークロードに高速の書き込みパフォーマンスが必要な場合は、高速の書き込み速度が適しています。

    • 書き込み速度を選択する方法を学ぶ

    • 既存のシステムで書き込み速度を変更する方法を学ぶ

    Kubernetesの永続ストレージとしてのオンプレミスONTAPクラスター

    Cloud Managerでは、オンプレミスのONTAPクラスターをコンテナーの永続ストレージとして使用できるようになりました。 Cloud Volumes ONTAPと同様に、Cloud ManagerはNetApp Tridentの展開を自動化し、ONTAPをKubernetesクラスターに接続します。

    KubernetesクラスターをCloud Managerに追加したら、Working EnvironmentsページからオンプレミスのONTAPクラスターに接続できます。

    始める方法を学びます 。

    Kubernetesの最新のトライデントバージョン

    Cloud Managerは、作業環境をKubernetesクラスターに接続すると、Tridentの最新バージョン（バージョン19.07.1）をインストールするようになりました。

    Azure汎用v2ストレージアカウントのサポート

    Azureに新しいCloud Volumes ONTAPシステムをデプロイすると、Cloud Managerが診断とデータ階層化のために作成するストレージアカウントが汎用v2ストレージアカウントになります。

    APIを使用したAzureストレージアカウント名のプレフィックス

    Cloud ManagerがCloud Volumes ONTAP用に作成するAzureストレージアカウントの名前にプレフィックスを追加できるようになりました。 Azureに新しいCloud Volumes ONTAPシステムをデプロイするときは、 storageAccountPrefixパラメーターを使用するだけです。

    APIの使用の詳細については、API開発者ガイドを参照してください 。

    Cloud Manager 3.7.4（2019年10月6日）

    Cloud Manager 3.7.4には、次の機能強化が含まれています。

    • Azure NetAppファイルのサポート

    • Cloud Volumes ONTAP for GCPの機能強化

    • S3へのバックアップの機能強化

    • AWSのブートディスクとルートディスクの暗号化

    • AWSバーレーンリージョンのサポート

    • Azure UAE North地域のサポート

    Azure NetAppファイルのサポート

    Cloud Managerから直接Azure NetApp FilesのNFSボリュームを表示および作成できるようになりました。この機能強化は、単一のインターフェースからクラウドストレージを管理できるようにするという当社の目標を引き継いでいます。

    始める方法を学びます 。

    この機能には、Azureの最新のCloud Managerポリシーに示されている新しいアクセス許可が必要です。 

     "Microsoft.NetApp/netAppAccounts/read", "Microsoft.NetApp/netAppAccounts/capacityPools/read", "Microsoft.NetApp/netAppAccounts/capacityPools/volumes/write", "Microsoft.NetApp/netAppAccounts/capacityPools/volumes/read", "Microsoft.NetApp/netAppAccounts/capacityPools/volumes/delete"

    Cloud Volumes ONTAP for GCPの機能強化

    Cloud Manager 3.7.4では、Cloud Volumes ONTAP for Google Cloud Platformに対して以下の機能強化が可能です。

    GCP Marketplaceでの従量制のサブスクリプション

    Google Cloud Platform MarketplaceでCloud Volumes ONTAPに登録すると、Cloud Volumes ONTAPの料金を支払うことができます。

    Google Cloud Platform Marketplace：Cloud Volumes ONTAPのCloud Manager

    共有VPC

    Cloud ManagerとCloud Volumes ONTAPは、Google Cloud Platform共有VPCでサポートされるようになりました。

    共有VPCを使用すると、複数のプロジェクトにわたる仮想ネットワークを構成および集中管理できます。 ホストプロジェクトで共有VPCネットワークをセットアップし、 サービスプロジェクトで Cloud ManagerおよびCloud Volumes ONTAP仮想マシンインスタンスをデプロイできます。 Google Cloudドキュメント：共有VPCの概要 。

    複数のGoogle Cloudプロジェクト

    Cloud Volumes ONTAPは、Cloud Managerと同じプロジェクトに存在する必要がなくなりました。 Cloud Managerサービスアカウントとロールを追加のプロジェクトに追加すると、Cloud Volumes ONTAPを展開するプロジェクトから選択できます。

    Cloud Managerサービスアカウントの設定の詳細については、このページの手順4bを参照してください 。

    Cloud Manager APIを使用する場合の顧客管理の暗号化キー

    Google Cloud Storageはデータをディスクに書き込む前に常に暗号化しますが、Cloud Manager APIを使用して、 顧客管理の暗号化キーを使用する新しいCloud Volumes ONTAPシステムを作成できます 。これらは、Cloud Key Management Serviceを使用してGCPで生成および管理するキーです。

    「GcpEncryption」パラメーターの使用の詳細については、 API開発者ガイドを参照してください。

    この機能には、GCPの最新のCloud Managerポリシーに示されている新しい権限が必要です 。 

     - cloudkms.cryptoKeyVersions.useToEncrypt - cloudkms.cryptoKeys.get - cloudkms.cryptoKeys.list - cloudkms.keyRings.list

    S3へのバックアップの機能強化

    これで、既存のボリュームのバックアップを削除できます。以前は、削除されたボリュームのバックアップのみを削除できました。

    S3へのバックアップの詳細をご覧ください 。

    AWSのブートディスクとルートディスクの暗号化

    AWS Key Management Service（KMS）を使用してデータ暗号化を有効にすると、Cloud Volumes ONTAPのブートディスクとルートディスクも暗号化されるようになりました。これには、HAペアのメディエーターインスタンスのブートディスクが含まれます。ディスクは、作業環境の作成時に選択したCMKを使用して暗号化されます。

    ブートおよびルートディスクは、AzureおよびGoogle Cloud Platformでは常に暗号化されます。これらのクラウドプロバイダーでは暗号化がデフォルトで有効になっているためです。

    AWSバーレーンリージョンのサポート

    Cloud ManagerおよびCloud Volumes ONTAPは、AWS中東（バーレーン）リージョンでサポートされるようになりました。

    Azure UAE North地域のサポート

    Cloud ManagerとCloud Volumes ONTAPがAzure UAE Northリージョンでサポートされるようになりました。

    サポートされているすべての地域を表示します 。

    Cloud Manager 3.7.3アップデート（2019年9月15日）

    Cloud Managerでは、Cloud Volumes ONTAPからAmazon S3にデータをバックアップできるようになりました。

    S3へのバックアップ

    S3へのバックアップは、Cloud Volumes ONTAPのアドオン機能であり、保護のための完全に管理されたバックアップおよび復元機能、およびクラウドデータの長期アーカイブを提供します。バックアップは、短期的なリカバリまたはクローン作成に使用されるボリュームSnapshotコピーとは無関係に、S3オブジェクトストレージに保存されます。この機能は、NetAppのクラウドバックアップサービスによって強化されています。

    始める方法を学びます 。

    この機能には、 Cloud Managerポリシーの更新が必要です。次のVPCエンドポイントのアクセス許可が必要になりました。 

     "ec2:DescribeVpcEndpoints", "ec2:CreateVpcEndpoint", "ec2:ModifyVpcEndpoint", "ec2:DeleteVpcEndpoints"

    Cloud Manager 3.7.3（2019年9月11日）

    Cloud Manager 3.7.3には、次の機能強化が含まれています。

    • AWS向けCloud Volumesサービスの発見と管理

    • AWS Marketplaceで新しいサブスクリプションが必要です

    • AWS GovCloudのサポート（米国東部）

    AWS向けCloud Volumesサービスの発見と管理

    Cloud Managerにより、AWSサブスクリプションのCloud Volumesサービスでクラウドボリュームを検出できるようになりました。検出後、Cloud Managerから直接クラウドボリュームを追加できます。この機能強化により、NetAppクラウドストレージを管理できる単一のペインが提供されます。

    始める方法を学びます 。

    AWS Marketplaceで新しいサブスクリプションが必要です

    AWS Marketplaceで新しいサブスクリプションを利用できます 。この1回限りのサブスクリプションは、Cloud Volumes ONTAP 9.6 PAYGOを展開するために必要です（30日間の無料試用システムを除く）。このサブスクリプションにより、Cloud Volumes ONTAP PAYGOおよびBYOLのアドオン機能を提供することもできます。作成したすべてのCloud Volumes ONTAP PAYGOシステムおよび有効にした各アドオン機能について、このサブスクリプションから請求されます。

    バージョン9.6以降、この新しいサブスクリプションメソッドは、以前にサブスクライブしたCloud Volumes ONTAP PAYGOの2つの既存のAWS Marketplaceサブスクリプションを置き換えます。 Cloud Volumes ONTAP BYOLをデプロイするときは、 既存のAWS Marketplaceページからサブスクリプションが必要です 。

    各AWS Marketplaceページの詳細をご覧ください 。

    AWS GovCloudのサポート（米国東部）

    Cloud ManagerとCloud Volumes ONTAPがAWS GovCloud（米国東部）リージョンでサポートされるようになりました。

    GCPでのCloud Volumes ONTAPの一般提供（2019年9月3日）

    Cloud Volumes ONTAPは、独自のライセンス（BYOL）を持ってくると、Google Cloud Platform（GCP）で一般的に利用できるようになりました。従量課金制のプロモーションも利用できます。このプロモーションでは、無制限の数のシステムの無料ライセンスを提供しており、2019年9月末に有効期限が切れます。

    • GCPの開始方法を学ぶ

    • サポートされている構成を表示する

    Cloud Manager 3.7.2（2019年8月5日）

    • FlexCacheライセンス

    • iSCSI用のKubernetesストレージクラス

    • iノードの管理

    • AWSでの香港地域のサポート

    • Azureのオーストラリア中部地域のサポート

    FlexCacheライセンス

    Cloud Managerは、すべての新しいCloud Volumes ONTAPシステムに対してFlexCacheライセンスを生成するようになりました。ライセンスには、500 GBの使用制限が含まれています。

    ライセンスを生成するには、Cloud Managerがhttps://ipa-signer.cloudmanager.netapp.comにアクセスする必要があります。このURLがファイアウォールからアクセス可能であることを確認してください。

    iSCSI用のKubernetesストレージクラス

    Cloud Volumes ONTAPをKubernetesクラスターに接続すると、Cloud ManagerはiSCSI Persistent Volumesで使用できる次の2つの追加Kubernetesストレージクラスを作成するようになりました。

    • netapp-file-san ：iSCSI Persistent VolumesをシングルノードCloud Volumes ONTAPシステムにバインドするため

    • netapp-file-redundant-san ：iSCSI Persistent VolumeをCloud Volumes ONTAP HAペアにバインドするため

    iノードの管理

    Cloud Managerは、ボリューム上のiノードの使用を監視するようになりました。 iノードの85％が使用されると、Cloud Managerはボリュームのサイズを増やして、使用可能なiノードの数を増やします。ボリュームに含めることができるファイルの数は、iノードの数によって決まります。

    Cloud Managerは、容量管理モードが自動に設定されている場合にのみiノードの使用状況を監視します（これはデフォルト設定です）。

    AWSでの香港地域のサポート

    Cloud ManagerとCloud Volumes ONTAPは、AWSのアジア太平洋（香港）リージョンでサポートされるようになりました。

    Azureのオーストラリア中部地域のサポート

    Cloud ManagerおよびCloud Volumes ONTAPは、次のAzureリージョンでサポートされるようになりました。

    • オーストラリア中部

    • オーストラリア中部2

    サポートされている地域の完全なリストを参照してください 。

    バックアップと復元に関する更新（2019年7月15日）

    3.7.1リリース以降、Cloud Managerはバックアップのダウンロードと、それを使用したCloud Manager構成の復元をサポートしなくなりました。 Cloud Managerを復元するには、次の手順に従う必要があります 。

    Cloud Manager 3.7.1（2019年7月1日）

    • このリリースには主にバグ修正が含まれています。

    • 1つの拡張機能が含まれています。CloudManagerは、NetAppサポートに登録されている各Cloud Volumes ONTAPシステム（新規システムと既存システムの両方）にNetApp Volume Encryption（NVE）ライセンスをインストールするようになりました。

      • NetAppサポートサイトアカウントをCloud Managerに追加する

      • 従量課金制システムの登録

      • NetApp Volume Encryptionのセットアップ

        Cloud Managerは、中国地域にあるシステムにNVEライセンスをインストールしません。

    Cloud Manager 3.7アップデート（2019年6月16日）

    Cloud Volumes ONTAP 9.6は、AWS、Azure、およびGoogle Cloud Platformでプライベートプレビューとして利用できるようになりました。プライベートプレビューに参加するには、 ng-Cloud-Volume-ONTAP-preview @ netapp.comにリクエストを送信します 。

    Cloud Volumes ONTAP 9.6の新機能をご覧ください

    Cloud Manager 3.7（2019年6月5日）

    • 今後のCloud Volumes ONTAP 9.6リリースのサポート

    • NetApp Cloud Centralアカウント

    • Cloud Backup Serviceを使用したバックアップと復元

    今後のCloud Volumes ONTAP 9.6リリースのサポート

    Cloud Manager 3.7には、今後のCloud Volumes ONTAP 9.6リリースのサポートが含まれています。 9.6リリースには、Google Cloud PlatformのCloud Volumes ONTAPのプライベートプレビューが含まれています。 9.6が利用可能になると、リリースノートを更新します。

    NetApp Cloud Centralアカウント

    クラウドリソースの管理方法を強化しました。各Cloud Managerシステムは、 NetApp Cloud Centralアカウントに関連付けられます 。このアカウントはマルチテナンシーを有効にし、将来、他のNetAppクラウドデータサービス用に計画されています。

    Cloud Managerでは、Cloud Centralアカウントは、Cloud Managerシステムと、ユーザーがCloud Volumes ONTAPを展開するワークスペースのコンテナです。

    Cloud Centralアカウントがマルチテナンシーを有効にする方法をご覧ください 。

    Cloud Managerは、Cloud Centralアカウントサービスに接続するためにhttps://cloudmanager.cloud.netapp.comにアクセスする必要があります。ファイアウォールでこのURLを開いて、Cloud Managerがサービスに接続できるようにします。

    システムとCloud Centralアカウントの統合

    Cloud Manager 3.7にアップグレードした後、NetAppは特定のCloud Managerシステムを選択してCloud Centralアカウントと統合します。このプロセス中に、NetAppはアカウントを作成し、各ユーザーに新しい役割を割り当て、ワークスペースを作成し、それらのワークスペースに既存の作業環境を配置します。 Cloud Volumes ONTAPシステムを中断することはありません。

    質問がある場合は、このFAQを参照してください 。

    Cloud Backup Serviceを使用したバックアップと復元

    NetApp Cloud Backup Service for Cloud Volumes ONTAPは、クラウドデータの保護と長期アーカイブのための完全に管理されたバックアップおよび復元機能を提供します。 Cloud Backup ServiceをCloud Volumes ONTAP for AWSと統合できます。サービスによって作成されたバックアップは、AWS S3オブジェクトストレージに保存されます。

    Cloud Backup Serviceの詳細をご覧ください 。

    開始するには、バックアップエージェントをインストールして構成し、バックアップおよび復元操作を開始します。ヘルプが必要な場合は、Cloud Managerのチャットアイコンを使用してお問い合わせください。

    この手動プロセスはサポートされなくなりました。 S3へのバックアップ機能は、3.7.3リリースでCloud Managerに統合されました。
