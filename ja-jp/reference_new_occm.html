    Cloud Manager 3.7の新機能 GitHubで編集 ドキュメントの変更をリクエストする

    08/05/2019 寄稿者

    RSS購読

    Cloud Managerは通常、毎月新しいリリースを導入して、新しい機能、拡張機能、およびバグ修正を提供します。

    以前のリリースをお探しですか？
    3.6の新機能
    3.5の新機能
    3.4の新機能

    Cloud Manager 3.7.2（2019年8月5日）

    • FlexCacheライセンス

    • iSCSI用のKubernetesストレージクラス

    • iノードの管理

    • AWSでの香港地域のサポート

    • Azureのオーストラリア中部地域のサポート

    FlexCacheライセンス

    Cloud Managerは、すべての新しいCloud Volumes ONTAPシステムに対してFlexCacheライセンスを生成するようになりました。ライセンスには、500 GBの使用制限が含まれています。

    ライセンスを生成するには、Cloud Managerがhttps://ipa-signer.cloudmanager.netapp.comにアクセスする必要があります。このURLがファイアウォールからアクセス可能であることを確認してください。

    iSCSI用のKubernetesストレージクラス

    Cloud Volumes ONTAPをKubernetesクラスターに接続すると、Cloud ManagerはiSCSI Persistent Volumesで使用できる次の2つの追加Kubernetesストレージクラスを作成するようになりました。

    • netapp-file-san ：iSCSI Persistent VolumeをシングルノードCloud Volumes ONTAPシステムにバインドするため

    • netapp-file-redundant-san ：iSCSI Persistent VolumeをCloud Volumes ONTAP HAペアにバインドするため

    iノードの管理

    Cloud Managerは、ボリューム上のiノードの数を監視し、必要に応じて最大数を増やします。ボリュームに含めることができるファイルの数は、iノードの数によって決まります。

    AWSでの香港地域のサポート

    Cloud ManagerとCloud Volumes ONTAPは、AWSのアジア太平洋（香港）リージョンでサポートされるようになりました。

    Azureのオーストラリア中部地域のサポート

    Cloud ManagerおよびCloud Volumes ONTAPは、次のAzureリージョンでサポートされるようになりました。

    • オーストラリア中部

    • オーストラリア中部2

    サポートされている地域の完全なリストを参照してください 。

    バックアップと復元に関する更新（2019年7月15日）

    3.7.1リリース以降、Cloud Managerはバックアップのダウンロードと、それを使用したCloud Manager構成の復元をサポートしなくなりました。 Cloud Managerを復元するには、次の手順に従う必要があります 。

    Cloud Manager 3.7.1（2019年7月1日）

    • このリリースには主にバグ修正が含まれています。

    • 1つの拡張機能が含まれています。CloudManagerは、NetAppサポートに登録されている各Cloud Volumes ONTAPシステム（新規システムと既存システムの両方）にNetApp Volume Encryption（NVE）ライセンスをインストールするようになりました。

      • NetAppサポートサイトアカウントをCloud Managerに追加する

      • 従量課金制システムの登録

      • NetApp Volume Encryptionのセットアップ

        Cloud Managerは、中国地域にあるシステムにNVEライセンスをインストールしません。

    Cloud Manager 3.7アップデート（2019年6月16日）

    Cloud Volumes ONTAP 9.6は、AWS、Azure、およびGoogle Cloud Platformでプライベートプレビューとして利用できるようになりました。プライベートプレビューに参加するには、 ng-Cloud-Volume-ONTAP-preview @ netapp.comにリクエストを送信します 。

    Cloud Volumes ONTAP 9.6の新機能をご覧ください

    Cloud Manager 3.7（2019年6月5日）

    • 今後のCloud Volumes ONTAP 9.6リリースのサポート

    • NetApp Cloud Centralアカウント

    • Cloud Backup Serviceを使用したバックアップと復元

    今後のCloud Volumes ONTAP 9.6リリースのサポート

    Cloud Manager 3.7には、今後のCloud Volumes ONTAP 9.6リリースのサポートが含まれています。 9.6リリースには、Google Cloud PlatformのCloud Volumes ONTAPのプライベートプレビューが含まれています。 9.6が利用可能になると、リリースノートを更新します。

    NetApp Cloud Centralアカウント

    クラウドリソースの管理方法を強化しました。各Cloud Managerシステムは、 NetApp Cloud Centralアカウントに関連付けられます 。このアカウントはマルチテナンシーを有効にし、将来、他のNetAppクラウドデータサービス用に計画されています。

    Cloud Managerでは、Cloud Centralアカウントは、Cloud Managerシステムと、ユーザーがCloud Volumes ONTAPを展開するワークスペースのコンテナです。

    Cloud Centralアカウントがマルチテナンシーを有効にする方法をご覧ください 。

    Cloud Managerは、Cloud Centralアカウントサービスに接続するためにhttps://cloudmanager.cloud.netapp.comにアクセスする必要があります。ファイアウォールでこのURLを開いて、Cloud Managerがサービスに接続できるようにします。

    システムとCloud Centralアカウントの統合

    Cloud Manager 3.7にアップグレードした後、NetAppは特定のCloud Managerシステムを選択してCloud Centralアカウントと統合します。このプロセス中に、NetAppはアカウントを作成し、各ユーザーに新しい役割を割り当て、ワークスペースを作成し、それらのワークスペースに既存の作業環境を配置します。 Cloud Volumes ONTAPシステムに混乱はありません。

    質問がある場合は、このFAQを参照してください 。

    Cloud Backup Serviceを使用したバックアップと復元

    NetApp Cloud Backup Service for Cloud Volumes ONTAPは、クラウドデータの保護と長期アーカイブのための完全に管理されたバックアップおよび復元機能を提供します。 Cloud Backup ServiceをCloud Volumes ONTAP for AWSと統合できます。サービスによって作成されたバックアップは、AWS S3オブジェクトストレージに保存されます。

    Cloud Backup Serviceの詳細をご覧ください 。

    開始するには、バックアップエージェントをインストールして構成し、バックアップおよび復元操作を開始します。ヘルプが必要な場合は、Cloud Managerのチャットアイコンを使用してお問い合わせください。
