    Cloud Centralのアカウント GitHubで編集 ドキュメントの変更をリクエストする

    06/08/2019 貢献者

    各Cloud Managerシステムは、 NetApp Cloud Centralアカウントに関連付けられています 。 Cloud Centralアカウントは、Cloud Managerシステムと、ユーザーがCloud Volumes ONTAPを展開するワークスペースのコンテナーです。

    Cloud Centralアカウントでマルチテナンシーが有効になります。

    • 単一のCloud Centralアカウントには、さまざまなビジネスニーズに対応する複数のCloud Managerシステムを含めることができます。

      ユーザーはCloud Centralアカウントに関連付けられているため、個々のCloud Managerシステムごとにユーザーを設定する必要はありません。

    • 各Cloud Managerシステム内で、複数のユーザーがCloud Volumes ONTAPシステムをワークスペースと呼ばれる独立した環境に展開して管理できます。

      これらのワークスペースは、共有されていない限り、他のユーザーには見えません。

    Cloud Managerをデプロイするときに、システムに関連付けるCloud Centralアカウントを選択します。

    アカウント管理者は、ユーザー、ワークスペース、およびサービスコネクタを管理することによって、このアカウントの設定を変更できます。

    詳しい手順については、Cloud Centralアカウントの設定を参照してください。

    Cloud ManagerがCloud Centralアカウントサービスに接続するには、 https：//cloudmanager.cloud.netapp.comにアクセスする必要があります。 Cloud Managerがサービスに連絡できるように、ファイアウォールのこのURLを開いてください。

    ユーザー、ワークスペース、およびサービスコネクタ

    Cloud Managerの[アカウント設定]ウィジェットを使用すると、アカウント管理者はCloud Centralアカウントを管理できます。アカウントを作成したばかりの場合は、最初から始めます。ただし、アカウントをすでに設定している場合は、そのアカウントに関連付けられているすべてのユーザー、ワークスペース、およびサービスコネクタが表示されます。

    ユーザー

    これらは、Cloud Centralアカウントに関連付けるNetApp Cloud Centralユーザーです。ユーザーとそのアカウント内の1つ以上のワークスペースを関連付けることで、それらのユーザーはCloud Managerで作業環境を作成および管理できます。

    ユーザを関連付けるとき、それらに役割を割り当てます：

    • アカウント管理者 ：Cloud Managerで任意の操作を実行できます。

    • Workspace Admin ：割り当てられたワークスペースでリソースを作成および管理できます。

    ワークスペース

    Cloud Managerでは、ワークスペースは他の作業環境から任意の数の作業環境を分離します。アカウント管理者が管理者をそのワークスペースに関連付けない限り、ワークスペース管理者はワークスペース内の作業環境にアクセスできません。

    作業環境はストレージシステムを表します。

    • 単一ノードのCloud Volumes ONTAPシステムまたはHAペア

    • ネットワーク内のオンプレミスのONTAPクラスタ

    • NetApp専用ストレージ構成内のONTAPクラスター

    サービスコネクタ

    サービスコネクタはCloud Managerの一部です。接続する少数のCloud Centralサービス（auth0およびCloud Centralアカウント）を除いて、Cloud Managerソフトウェアの大部分（ユーザーインターフェースなど）を実行します。サービスコネクタは、クラウドプロバイダにデプロイされた仮想マシンインスタンス、または構成したオンプレミスホストで実行されます。

    サービスコネクタは、複数のNetAppクラウドデータサービスと共に使用できます。たとえば、Cloud Manager用のサービスコネクタがすでにある場合は、Cloud Tieringサービスを設定するときにそれを選択できます。

    例

    次の例は、2つのワークスペースを使用してCloud Volumes ONTAPシステム用の分離環境を作成するアカウントを示しています。たとえば、一方のワークスペースがステージング環境用で、もう一方のワークスペースが本番環境用であるとします。

    Cloud ManagerとCloud VolumesのONTAPシステムは、実際にはNetApp Cloud Centralアカウントには存在せず、クラウドプロバイダーで実行されています。これは、各コンポーネント間の関係を概念的に表したものです。

    2つの別々のCloud Centralアカウントを使用して最高レベルのマルチテナントを示す別の例を次に示します。たとえば、サービスプロバイダーは1つのCloud CentralアカウントでCloud Managerを使用して顧客にサービスを提供しながら、別のアカウントを使用して1つの事業部門に障害回復を提供することができます。

    Cloud Centralアカウントとの統合に関するよくある質問

    Cloud Manager 3.7にアップグレードした後しばらくして、NetAppはCloud Centralアカウントと統合するために特定のCloud Managerシステムを選択します。このFAQはあなたがプロセスに関して持つかもしれない質問に答えることができます。

    このプロセスにはどのくらいの時間がかかりますか？

    ほんの数分。

    Cloud Managerは利用できませんか？

    いいえ、それでもCloud Managerシステムにアクセスできます。

    Cloud Volumes ONTAPについてはどうですか？

    Cloud Volumes ONTAPシステムに支障はありません。

    この過程で何が起こりますか？

    NetAppは、統合プロセス中に次のことを行います。

    1. 新しいCloud Centralアカウントを作成し、それをCloud Managerシステムに関連付けます。

    2. 既存の各ユーザーに新しい役割を割り当てます。

      • Cloud Manager管理者はアカウント管理者になります

      • テナント管理者および作業環境管理者はワークスペース管理者になります

    3. 既存のテナントを置き換えるワークスペースを作成します。

    4. あなたの職場環境をそれらのワークスペースに置きます。

    5. サービスコネクタをすべてのワークスペースに関連付けます。

    Cloud Managerシステムをどこにインストールしたかは重要ですか？

    いいえ。NetAppは、AWS、Azure、または構内にかかわらず、システムがどこにあるかに関係なく、Cloud Centralアカウントとシステムを統合します。
