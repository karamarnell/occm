    GCPアカウントをセットアップしてCloud Managerに追加する GitHubで編集 ドキュメントの変更をリクエストする

    09/03/2019 寄稿者

    Cloud Volumes ONTAPシステムでデータ階層化を有効にするには、Cloud AdminにStorage Admin権限を持つサービスアカウントのストレージアクセスキーを提供する必要があります。 Cloud Managerはアクセスキーを使用して、データ階層化のためのCloud Storageバケットをセットアップおよび管理します。

    Google Cloud Storageのサービスアカウントとアクセスキーを設定する

    サービスアカウントにより、Cloud Managerはデータ階層化に使用されるCloud Storageバケットを認証およびアクセスできます。キーは、Google Cloud Storageが誰がリクエストを行っているかを知るために必要です。

    手順

    1. GCP IAMコンソールを開き、 Storage Adminロールを持つサービスアカウントを作成します 。

    2. GCPストレージ設定に移動します。

    3. プロンプトが表示されたら、プロジェクトを選択します。

    4. [ 相互運用性 ]タブをクリックします。

    5. まだ行っていない場合は、[ 相互運用性アクセスを有効にする]をクリックします 。

    6. [ サービスアカウントのアクセスキー ]で、[ サービスアカウントのキーの 作成 ]をクリックします 。

    7. 手順1で作成したサービスアカウントを選択します。

    8. キーの作成をクリックします 。

    9. アクセスキーとシークレットをコピーします。

      データ階層化用のGCPアカウントを追加するときに、Cloud Managerでこの情報を入力する必要があります。

    GCPアカウントをCloud Managerに追加する

    サービスアカウントのアクセスキーを取得したので、それをCloud Managerに追加できます。

    手順

    1. Cloud Managerコンソールの右上にある[設定]アイコンをクリックし、[ クラウドプロバイダーとサポートアカウント]を選択します 。

    2. [ 新しいアカウントを追加]をクリックし、[ GCP ]を選択します。

    3. サービスアカウントのアクセスキーとシークレットを入力します。

      これらのキーにより、Cloud Managerはデータ階層化のためにCloud Storageバケットを設定できます。

    4. ポリシー要件が満たされていることを確認し、[ アカウントの作成]をクリックします 。

    次は何ですか？

    ボリュームを作成、変更、または複製するときに、個々のボリュームでデータ階層化を有効にできるようになりました。詳細については、 非アクティブなデータを低コストのオブジェクトストレージに階層化するを参照してください。

    ただし、その前に、Cloud Volumes ONTAPが存在するサブネットがプライベートGoogleアクセス用に構成されていることを確認してください。手順については、 Google Cloudドキュメント：プライベートGoogleアクセスの設定を参照してください 。
